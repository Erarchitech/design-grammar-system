# 2026-06-01 Ontology v6.0 Restructure — Core Band, SpecGraph, Partonomy

## Context

Full ontology restructure based on the revised ERD (sheet **02_TAXONOMY** in `ontology/900_04_Ontologies_Rev06.drawio`). The ERD introduced a **Core** over-layer band of main entities sitting above the five graph layers, renamed the Knowledge layer to SpecGraph, moved Reasoner into the ValidationGraph, and specified explicit part-whole containment for all ERD groupings.

### 3. Structural changes (`DesignGrammar-V6.owl`)

#### New `dg:Core` top-level class (over-layer — no `dg:graph` tag)
- `dg:Object` (was `dgc:Object`) — has Function, Behavior, Structure
- `dg:Function` (was `dgc:Function`) — covered by OntoGraph; evaluatedBy Reasoner
- `dg:Behavior` (was `dgc:Behavior`) — hasAlgorithm → ComputationGraph
- `dg:Structure` (was `dgc:Structure`) — hasGeometry, hasTopology
- `dg:Geometry` (was `dgc:Geometry`) — part of Structure
- `dg:Topology` (was `dgc:Topology`) — part of Structure; anchor for BOT/Topologic extensions
- `dg:DesignState` (was `dgv:DesignState`) ⊃ {`dg:ObjectState`, `dg:DefState`}
- `dg:Session` ← NEW: unifies `dgm:DesignRuleSession` + `dgk:KnowledgeSession`

#### IRI shortening
- `metagraph#` → `meta#`
- `validation#` → `valid#`
- `computation#` → `comp#`
- `knowledge#` → `spec#` (+ `dgk`→`dgs`, Knowledge*→Spec*)

#### Layer changes
- **SpecGraph** (`dgs:`): `KnowledgeGraph/Class/Note/Tag → SpecGraph/SpecClass/SpecNote/SpecTag`; `KnowledgeSession` removed (→ Core `Session`)
- **Metagraph**: `DesignRuleSession` removed (→ Core `Session`); `Reasoner` removed (→ ValidationGraph)
- **ValidationGraph**: `Reasoner` moved in — corresponds to the Grasshopper "Validator" component; `DesignState/DefState/ObjectState` moved out (→ Core)
- **ComputationGraph**: `Object/Function/Behavior/Structure/Geometry/Topology` moved out (→ Core); `ParametricState` dropped

#### New object properties
- `dg:objectRef` (ObjectState → Object) — the ERD "ObjectRef" edge
- `dg:definesFunction` (OntoGraph entity → Function) — OntoGraph defines Functions of Objects
- `dgv:refersToGeometry` (GeoRef → Geometry) — completes the ObjectState→GeoRef→Geometry chain
- `dgm:validates` (Rule → DesignState) — Metagraph validates Design State
- `dgv:objectRefName` — renamed from `dgv:objectRef` (the SWRL variable-name *string*)

### 4. ERD partonomy (part-of hierarchy)

Added a transitive part-whole backbone `dg:hasPart` (inverse `dg:partOf`). The 10 ERD composition properties are declared `rdfs:subPropertyOf dg:hasPart`:

| Grouping | Property |
|---|---|
| Object → Function | `dg:hasFunction` |
| Object → Behavior | `dg:hasBehavior` |
| Object → Structure | `dg:hasStructure` |
| Structure → Geometry | `dg:hasGeometry` |
| Structure → Topology | `dg:hasTopology` |
| RuleSet → Rule | `dgm:hasRule` |
| Algorithm → Procedure | `dgc:hasProcedure` |
| Procedure → Pattern | `dgc:hasPattern` |
| Pattern → Parameter | `dgc:hasParameter` |
| Pattern/Procedure → Interface | `dgc:hasInterface` |

Transitivity of `dg:hasPart` enables full decomposition by inference (e.g. Object hasPart Geometry). Disjointness and is-a semantics are preserved — this is a **partonomy** (part-of), not subclass nesting. `dgv:hasParameter` (DesignState→DesignStateParameter) is intentionally excluded.

## Key Decisions
- `ParametricState` dropped — state represented by Core `dg:DesignState`
- Scope: ontology files only (runtime — Neo4j labels, n8n, data-service — out of scope)
- Full re-namespace: Core entities in `dg:`; Knowledge→Spec gets `spec#`
- Partonomy approach: rigorous part-of via `dg:hasPart` sub-property hierarchy (not subClassOf nesting)
