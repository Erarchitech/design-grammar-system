### Key Decision: Monochromatic Verdict Reconciliation (DSYS-01)

The CONTEXT's shorthand describes verdicts as "green" (Consistent), "amber" (Unknown), "red" (Inconsistent) — but the design system is **strictly monochromatic** per DSYS-01: *"Color is absence, not expression. One chromatic hue exists — Signal Red."* The palette offers no green or amber token, and DSYS-01 forbids inventing one.

**Solution locked in UI-SPEC, Dimension 3 (Color):**

Four verdict states told apart by **weight, opacity, glyph, copy** — never hue:

| Verdict | Hue | Tokens | Glyph | Badge | Distinguisher |
|---------|-----|--------|-------|-------|---------------|
| Consistent | achromatic (NOT green) | `--color-ink`, `Badge variant="solid"` | `✓` | "Consistent" (solid) | Strong ink, weight 600 |
| Inconsistent | **Signal Red** (the one hue) | `--color-signal`, `Badge variant="signal"` | — | "Inconsistent" (signal) | Count + labeled class list |
| Unknown | achromatic (NOT amber) | `--text-muted` (#737373), `Badge variant="outline"` | `–` | "Inconclusive" (outline) | Mid-gray + "timed out" copy |
| Cancelled | achromatic, quietest | `--ink-a56` | `×` | none | Faint, no badge, quiet copy |

This aligns with `colors.css` semantics: `--status-fail` (Signal Red) for Inconsistent, `--status-pass` (ink) for Consistent, `--status-base` (mid-gray) for Unknown/Cancelled.

### Secondary Finding: Phantom Tokens

The current `ReasonerScreen.jsx` references **`--color-accent`** (lines 176, 227) and **`--surface-focus`** (line 179), which are **undefined** anywhere in `ui-v2/src/styles/` — the selection ring/wash silently resolve to nothing today. The CONTEXT's discretion note lists `--color-accent` as "existing," but it doesn't. The UI-SPEC documents this as an in-scope cleanup: any selection/accent styling the phase edits must use real tokens (`--accent-selection` / `--color-signal`) instead.
