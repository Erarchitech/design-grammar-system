## Key decisions (user-confirmed)

- **v3.0 superseded**, not absorbed or left open. Phase 7 foundation (VariableKind, VariableTypeInferrer, DesignStateIdGenerator) carries forward; Phases 8ŌĆō12 dropped.
- **Full ontology rename V6ŌåÆV7** to match the schema's notation exactly (`Ontograph`, `Validgraph`, `Computgraph`, `ObjProperty`, `DataProperty`, `Run`, `ObjState`, `ReStatus`, new `ParamState`/`PropState`). Justification: V6 has zero runtime consumers, exists only for the PhD publication series ŌĆö safe to rename. Guarded by a `V6-to-V7-mapping.md` recovery file (old IRI ŌåÆ new IRI) so published papers can still be cross-referenced.
- **SpecGraph runtime rename included in v7.0 scope** (not deferred) ŌĆö closes the drift found in step 3 above. Full pass: data-service, n8n knowledge workflows, NeoVis, UI, DB migration script.
- **No GSD research step** ŌĆö this is an internal restructuring driven by the user's own schema; research agents would add no signal.
- Component-port policy: **update where overlapping with the new schema, keep where no overlap** (e.g. VALIDATOR keeps `DataServiceUrl` input and `Report`/`FailingBindings`/`ValidationRunId`/`ModelViewerUrl` outputs ŌĆö none of these appear in the schema but none contradict it either).
