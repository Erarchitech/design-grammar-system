# v8.0 Phases 21–26: Autonomous Execution + Live Verification

## Context

`/gsd-execute-phase` invoked for the whole v8.0 milestone (phases 21–26)
autonomously, with architectural decisions taken at Claude's discretion and
GSD planning artifacts (PLAN/SUMMARY) written post facto rather than
up-front. Goal: replace the legacy dark SPA with the V2 light UI as the
`design-grammars` container.

## What shipped

- **Phase 21 — Design System Foundation:** new `ui-v2/` Vite+React app;
  6-file token CSS split copied from `design/v2/_ds/`; Geist/Geist
  Mono/Oswald self-hosted via `@fontsource` (no CDN); all ~23 component
  recipes reconstructed as clean JSX from the compiled `_ds_bundle.js`
  (original `.jsx` sources don't exist on disk — bundle-only); dev-only
  specimen page at `/#specimen`.
- **Phase 22 — Shell, Landing, Auth:** landing particle-ring engine ported
  from the design mockup's canvas logic (10,400 particles, gaussian peaks
  toward region anchors, hero title materialise/dissolve); 520ms fly
  transitions between 4 always-mounted layers; auth reuses the **legacy
  `dg_users`/`dg_current_user` localStorage contract** byte-for-byte —
  existing accounts survive the cutover.
- **Phase 23 — Graph Viewer:** live orbital datascape — one ring layer per
  Neo4j `graph` property value, 3 label orbits per layer, divergence field,
  spire callouts, minimap; prompt console wired to the real n8n webhooks.
  **Found and fixed two live backend bugs** — see
  [[debugging/n8n workflows had a fatal quote bug and silent project-scoping failure|debugging note]].
- **Phase 24 — Model Viewer:** validation runs from data-service, grouped by
  design state; iso map (deterministic synthetic massing — true Speckle 3D
  deferred, see REQUIREMENTS Future); SWRL rule panel from the metagraph.
  Seeded `test/seed_validation_run.cypher` (project `v8-ui-smoke`) since real
  runs only publish from Grasshopper and pre-v4 runs are migration-blocked.
- **Phase 25 — Projects:** live project tile grid; scoping verified by
  switching projects and diffing both viewers' data.
- **Phase 26 — Cutover:** `docker-compose.yml` `design-grammars.build` now
  points at `./ui-v2`; 7-point E2E parity checklist passed against the live
  stack (serving, config injection, graph browse, validation browsing,
  LLM query, rule ingest, scoping). `docs/RELEASE-NOTES-v8.0.md` written.
