## Key findings

1. **GSD audit limitation:** UAT files using `expected: |` block scalars are silently dropped. The `query audit-uat` regex (line 258 of uat.cjs) requires single-line `expected:` immediately followed by `result:`. This affects 28-UAT and 29-UAT too (both already outside audit view). **Fix:** use inline `expected:` in all future `## Tests` sections.

2. **Phase 29 root cause fully resolved:** The "no design states found" gap from v8.2 was caused by dg_context.py feeding the LLM an aspirational Validgraph schema (DesignState/Run nodes) that doesn't exist in production (only ValidationRun nodes with statePayloadJson blob exist). Phase 29 gap-closure plans 29-06/29-07 fixed this by converging the schema and wiring existing_design_states() into assemble_context(). Live re-run 2026-07-20 confirms both the routing path and the grounded answer.

3. **v9.0 pipeline is fundamentally one test run, not nine:** The 32â†’33â†’34â†’35â†’36â†’37 canvas chain has such tight coupling (each output feeds the next) that testing them sequentially means a single Group 4 E2E run transitively proves 5 upstream phases.
