## Bug: get_adapter() ignored baseUrl

**Root cause:** `get_adapter("openai")` always created `OpenAIAdapter()` with default `base_url="https://api.openai.com/v1"`, ignoring `settings.get("baseUrl")`. DeepSeek API key was sent to OpenAI servers → 401.

**Fix:** Threaded `base_url` parameter through all layers:
- `get_adapter(provider, base_url)` → `OpenAIAdapter(base_url=...)`
- `list_models_for_provider(provider, api_key, base_url)`
- Both `llm_generate` and `test_llm_settings` pass `settings.get("baseUrl")`

See [[debugging/get_adapter ignores baseUrl from settings]].
