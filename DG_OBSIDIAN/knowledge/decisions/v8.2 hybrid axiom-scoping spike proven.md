# v8.2 Hybrid axiom-scoping: static TBox ∪ live export + curated disjointness

**Decision:** OWL reasoning in v8.2 runs against a **hybrid** ontology: the static `DesignGrammar-V7.owl` TBox (69 subClassOf, 105 domain, 112 range, 235 owl:Class declarations) is unioned at reasoning time with a project-scoped live export of the Neo4j OntoGraph/Metagraph, plus curated `owl:disjointWith` axioms seeded into the static TBox. Extending the LLM ingestion pipeline to emit real axioms is explicitly deferred out of v8.2.

**Alternatives considered:**
1. **Extend LLM ingestion** — modify n8n prompts and Cypher templates to emit `subClassOf`/`domain`/`range`/`disjointWith` axioms on each rule ingest. Rejected: requires 9-file schema propagation (CLAUDE.md list), mid-milestone n8n prompt churn, and would still have zero disjointness axioms for existing rules.
2. **Scope reasoning to structural/referential checks only** — no axiom curation needed, but produces an always-pass result that architects learn to distrust. Rejected: the milestone explicitly requires meaningful consistency checking.
3. **Hybrid (chosen)** — union the already-rich static V7 TBox with live export + curated disjointness.

**Spike evidence (empirical):**
- **Naive flat export of `v8-ui-smoke` → HermiT:** 0 inconsistent classes. The flat export has no axioms connecting domain terms — trivially "consistent". *This is the documented false positive.*
- **Hybrid + `ex:Door owl:disjointWith ex:Window` + seeded `ex:SlidingDoor`:** 2 inconsistent classes (`owl.Nothing`, `ex.SlidingDoor`). HermiT caught the TBox contradiction in 1.23s.
- **Axiom counts:** DesignGrammar-V7.owl has 65 subClassOf, 101 domain, 110 range, **0 disjointWith** — the empirical justification for curated disjointness.

**Consequences:**
- Phase 821's exporter must implement the hybrid union (static TBox + live export)
- A one-time design-activity pass to curate disjointness axioms into the static TBox is needed before reasoning on real client projects
- The naive-export false-positive and hybrid-truth evidence are captured in `820-DECISION.md` for future reference
