# T1 — Онтологический фреймворк для дизайн-грамматик

## Полное название

**REINVENTING DESIGN GRAMMARS BASED ON ONTOLOGICAL FRAMEWORK FOR INTERACTIVE PARAMETRIC DESIGN VALIDATION**

## Аннотация (краткая)

Представляет двухслойную граф-схему (OntoGraph + Metagraph) для хранения SWRL-правил в Neo4j. Описывает семантику нарушений (violation-pattern): тело правила кодирует условие нарушения, голова — булевый флаг. Оценивает точность LLM-кодирования (95%) на выборке из 40 правил.

## Ключевые вклады

1. Двухслойная схема: OntoGraph (Class, DatatypeProperty, ObjectProperty) + Metagraph (Rule, Atom, Var, Literal, Builtin)
2. Семантика SWRL-нарушений с инвертированной логикой (тело срабатывает при нарушении)
3. Конвенция Rule_Id: `R_<DOMAIN>_<PROPERTY>_<LIMIT>_V`
4. Оценка точности LLM-кодирования: 95% (38/40 правил корректно)
5. Среднее время авторинга: ~45 мин на правило (baseline)

## Основные технические детали

- Узлы OntoGraph: `Class`, `DatatypeProperty`, `ObjectProperty`
- Узлы Metagraph: `Rule`, `Atom`, `Var`, `Literal`, `Builtin`
- Отношения: `HAS_BODY`, `HAS_HEAD`, `REFERS_TO`, `ARG` (с порядковым pos)
- Atom_Id: `{Rule_Id}_A{n}` (тело), `{Rule_Id}_H{n}` (голова)
- Семантическое отображение: "максимум X" → `swrlb:greaterThan(?val, X)`
