# T2 — Кодирование правил и интеграция в параметрический воркфлоу

## Полное название

**Design Rule Encoding and Integration in Parametric Design Workflow**

## Аннотация (краткая)

Описывает n8n-воркфлоу из 17 узлов для конвертации NL-правил в SWRL/Cypher через LLM (llama3.1, temp=0.1). Документирует восемь компонентов Grasshopper-плагина, алгоритм SWRL-парсера, инженерию промптов и edit-mode. Кейс-стади: фасад жилого здания, 9 правил, 2 состояния дизайна.

## Ключевые вклады (не пересекаются с T1)

1. 17-узловой n8n-воркфлоу: webhook → LLM → парсинг → MERGE в Neo4j → async polling
2. 5 компонентов LLM-промпта: ограничения схемы, семантическая таблица, директива разделения, IRI-инвентарь, NL-текст
3. Бespoke SWRL-парсер (C# regex): ClassAtom / DataPropertyAtom / BuiltinAtom по числу аргументов
4. MERGE-идемпотентность Cypher: все узлы создаются через MERGE с детерминированными Atom_Id
5. Edit-mode: ключевые слова (edit/update/modify) + Rule_Id → MATCH-DELETE → пересоздание
6. 8 компонентов Grasshopper: CONNECTOR, METAGRAPH, RULE DECONSTRUCT, CLASSIFICATOR, DESIGN STATE, VALIDATOR, REINSTATE, VALIDATION RUNS
7. ScheduleSolution async-паттерн: Task.Run + хеш ключа запроса + переоценка каждые 200 мс
8. DataTree-привязка CLASSIFICATOR: ветка i Values DataTree → переменная i списка Variables

## Не переопределяет из T1

- Схему OntoGraph/Metagraph (ссылка: Ermolenko, 2026a)
- Семантику SWRL-нарушений
- Конвенцию Rule_Id

## Кейс-стади

- Жилой фасад, 9 правил (площадь, угол юг, коэффициент остекления)
- 2 состояния дизайна, цикл редактирования ограничения
