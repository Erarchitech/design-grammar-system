SLIDE 13
  [TXT] Knowledge Graph | A structured way to organize data
  [TXT] A knowledge graph is an organized representation of information that models real-world entities and relationships between them. It maps data visually as a network of connected labeled entities with properties as nodes and edges, so the data keeps its meaning and context.
  [NOTES] an ontology example could be a system that defines building, room, door, window, and floor as concepts, then specifies relationships like “a room is part of a building” and “a door connects two spaces”

  [NOTES] TO sum up, This research aims to develop Design Grammars as A generative copilot system that connects architectural knowledge  | with ontological object relationships across domains, moving beyond manual file exchanges.  | It enables collecting parametric rules, synchronizing key parameters between platforms, smart design assistance,  | AI-driven insights and feedback, design validation, LLM communication, machine learning on design history, and dataset reuse.  | I believe, this approach supports more sustainable and consistent design planning.  | For future developments, I want to review ontology, develop API, develop modular database, so it handles different types of data efficiently dy dedicated modules, setup sunchronization..
