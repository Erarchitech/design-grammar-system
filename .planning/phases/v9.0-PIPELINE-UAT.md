# v9.0 — Single-Pipeline UAT Test Plan

> Consolidated human test plan for milestone **v9.0 AI Workflow Intelligence** (phases 28–40).
> Generated 2026-07-20 by `/gsd-audit-uat`. Supersedes ad-hoc per-phase testing where phases chain.

## Why one pipeline

The v9.0 canvas-intelligence phases (32 → 33 → 34 → 35 → 36 → 37) are a **single continuous flow on
one shared fixture — the Frame definition**. Each phase's output is the next phase's input, so a later
phase's success criterion **transitively re-proves** the earlier phases. Testing per-phase would test the
same behavior up to 4 times. This plan runs the chain **once** and adds only the negative / robustness /
aesthetic checks a happy path can't reach.

Two independent tracks:
- **Track A** (browser + Docker, no Rhino): 28 → 29 — the LLM/cloud chain.
- **Track B** (Rhino + Docker): 33 → 34 → 35 → 36 → 37 — the canvas chain, all on the Frame definition.

## GROUP 4 — The main E2E spine (Track B, Rhino + Docker) — covers 35, 36, transitively 32+33+34

*Run on the Frame definition with **only** the Object marker + 2 Proc tags present as anchors.*

### 4.1 Recognition (35 test 1 · transitively 33 SC1/SC2, 29 catalog)
- `POST /computgraph/recognize` → proposals for patterns/parameters/interfaces whose member sets match the
  reference annotation for ≥ majority of blocks, each with confidence + rationale; unclassifiable blocks in
  `unrecognized[]`. *(Bridge pull + MCP + gateway all exercised here.)*

### 4.2 Preview render + undo cleanliness (35 test 2/6)
- `preview_structure` draws desaturated `[?] <name> (<confidence>%)` groups + one scribble legend; a
  **single Ctrl+Z** removes every trace; `clear_preview` leaves no residue.
- Run `preview_structure` twice (2nd auto-clears 1st); two Ctrl+Z walk both records without orphans.

### 4.3 Concurrent-solve safety (35 test 3)
- Trigger `preview_structure` mid-solve → no Rhino crash / doc corruption (InvokeOnCanvasWrite marshals correctly).

### 4.4 Accept / reject / mixed undo (35 test 4/5)
- **DG STRUCTURE CONFIRM**: accept some, reject some → single Ctrl+Z restores rejected previews and reverts
  accepted restyles coherently.
- Accepted groups get `source: recognized` (`dg.recognized.<guid>`) and **survive save + reopen**, parsing
  identically to hand tags.

### 4.5 Publish + idempotency + provenance (36 SC1/SC2/SC3 · transitively 32+34+35)
- **DG COMPUTGRAPH PUBLISH** the confirmed Frame → Neo4j subgraph: 1 Object–Behavior–Algorithm chain,
  2 Procedures, correct patterns/parameters (kinds)/interfaces, project-scoped. **← This single count/shape
  check re-proves 32 parse + 34 tags + 35 recognition were all correct.**
- Re-publish same definition → **zero node-count change** (MERGE idempotency).
- Provenance query returns `source / model / definitionId / publishedAt` per node; every node carries its `dgId`.

### 4.6 ui-v2 Computgraph layer display (36 SC4)
- Open the ui-v2 Graph screen → Computgraph layer shows distinctly-styled Object/Behavior/Algorithm/
  Procedure/Pattern/Parameter/Interface nodes, filterable per project. Hard-refresh (Ctrl+Shift+R) after
  any container rebuild.
