### 1. Collapsible Session History Panel

**Position:** Bottom of Specs&Notes tab content, below mode-specific panels (insert/query/update). Always rendered regardless of `knowledgeMode`. *(Source: D-01, D-03)*

**Header bar:**
- Height: `38px` (matches `.mode-trigger` height)
- Background: `#0b0f19` (`--control-bg`)
- Border: `1px solid #273041` top and bottom (`--control-border`)
- Border-radius: `8px` (matches controls)
- Padding: `0 14px`
- Display: `flex`, `align-items: center`, `justify-content: space-between`
- Cursor: `pointer` (entire header is clickable)
- Left side: Chevron SVG (▶ collapsed / ▼ expanded) + "Session History" label (`13px`, weight `700`, color `#f4f7fb`) + count badge
- Right side: Mode filter dropdown

**Count badge:**
- Inline next to label text, e.g. `Session History (12)`
- The count number uses color `#98a4bd` (muted) and weight `400`
- Parentheses included in the label string: `"Session History (" + count + ")"`

**Expand/collapse chevron:**
- Same SVG `<path>` pattern as `.mode-chevron` (existing `M6 9L12 15L18 9`)
- Size: `16px` × `16px`
- Color: `#98a4bd` (muted), hover: `#5fd0ff` (accent)
- Rotated: `transform: rotate(-90deg)` when collapsed, `rotate(0deg)` when expanded
- No transition animation (matches existing codebase — no CSS transitions used)

**Collapsed state:** Only header bar visible. *(Source: D-02)*

**Expanded state:** Header bar + session list below it, max-height `280px` with `overflow-y: auto`. Custom scrollbar matching existing sidebar pattern: `scrollbar-color: #a3a6af #1b202a`.

## Interaction Contracts

### Session Fetch

- **Trigger:** `useEffect` on Specs&Notes tab mount (extends existing `useEffect` at line 2770)
- **Endpoint:** `GET ${getDataServiceBaseUrl()}/knowledge/sessions/${project}`
- **Response shape:** `{ project, sessions: [{ sessionId, mode, prompt, result, createdAt }] }`
- **Storage:** Single `useState` holding full array, sorted reverse-chronological (server returns this order)
- **Error handling:** Silently fail (set empty array). No error toast for background fetch.
- **Re-fetch:** On every tab switch to Specs&Notes (not polling)

### Session Display Cap

- **Initial display:** Show first 50 sessions from the fetched array
- **"Show more" button:** Appears below the list if `sessions.length > displayedCount`
- **Button style:** Matches `.secondary-btn` pattern — transparent background, accent border+text, `12px` font, centered
- **Button text:** `"Show more"`
- **Increment:** +50 per click
- **No server-side pagination** — all sessions fetched in single call, capped client-side *(Source: D-11)*

### Restore Behavior

- Sets `knowledgePromptText` to `session.prompt` *(Source: D-07)*
- Sets `knowledgeResponseText` to `session.result` *(Source: D-07)*
- Calls `setKnowledgeMode(session.mode)` to switch dropdown *(Source: D-07)*
- Does NOT trigger any backend call *(Source: D-08)*
- Does NOT close the session history panel (panel stays expanded)
- Does NOT scroll the sidebar (user sees restored content by scrolling up themselves)

### Relative Date Formatting

Utility function `formatRelativeDate(isoString)`:

| Condition | Output |
|-----------|--------|
| < 1 minute ago | `"just now"` |
| < 60 minutes ago | `"Xm ago"` |
| < 24 hours ago | `"Xh ago"` |
| < 7 days ago | `"Xd ago"` |
| ≥ 7 days | `"YYYY-MM-DD"` (ISO short date) |

Implemented as a plain function (no library dependency). Computed at render time from `createdAt` field.
