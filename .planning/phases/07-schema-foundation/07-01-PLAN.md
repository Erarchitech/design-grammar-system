<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Rule.Swrl (Neo4j-sourced) → VariableTypeInferrer | SWRL expression text originates from LLM-generated Cypher (untrusted-ish, already persisted); VariableTypeInferrer consumes already-parsed `Atom`/`AtomArg` structures, not raw strings, so injection surface is limited to malformed atom data, not Cypher/SWRL text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-------------------|
| T-07-01 | Tampering | VariableTypeInferrer.Infer | accept | Pure in-memory function over already-validated `Rule`/`Atom` objects (no external I/O); malformed atom data can only come from `Neo4jRuleRepository` which is covered separately in 07-03-PLAN.md's threat model |
| T-07-02 | Information Disclosure | Variable.Kind nullable property | accept | Kind is computed transiently per-read and never persisted to Neo4j (per CONTEXT.md "NOT stored on Var nodes") — no new data-at-rest exposure surface introduced |
| T-07-03 | Denial of Service | VariableTypeInferrer.Infer called per-variable per-rule | accept | O(atoms × args) per call on already-bounded in-memory collections (rules per project are small); no unbounded recursion or external calls |
</threat_model>
