<objective>
Lock the `DefState`/`ObjectState`/`ObjectInstance` C# model hierarchy and the `DS_`/`OS_`/`OI_` ID-generation logic, plus the `IDR_` prefix constant, per CONTEXT.md's "DesignState Neo4j Hierarchy" and "ID Formats" sections (SCHM-01, SCHM-03, CMPST-06/07 groundwork). This plan delivers ONLY the C# model + ID-generation layer in `DG.Core` — no Neo4j Cypher (that lands in 07-03-PLAN.md), no Grasshopper component behavior changes (Phase 9 consumes these models later). SCHM-03 is partially covered here: DS_/OS_/OI_ are fully delivered (model classes + ComputeXxxId methods); IDR_ is declared as a prefix constant only — IdRef is DefState.StateId reused under a different name per CMPST-08 (not an independently-hashed entity), and its production wiring belongs to the DESIGN STATE component, which does not exist until Phase 9.

Purpose: `DefState`/`ObjectState`/`ObjectInstance` are the typed contracts every later v3.0 phase (9, 10, 11) wires into OBJECT STATE, DESIGN STATE, CLASSIFICATOR, and RUN DECONSTRUCT. The SHA256 ID-generation logic currently lives inside `DesignStateComponent.cs` behind a `#if GRASSHOPPER_SDK` guard — this plan relocates/extends that logic into `DG.Core` so it is unit-testable and reusable from non-GH code (Neo4j repository, future components) per the project's stated principle that `DG.Core` holds "pure logic (no GH deps)."

Output: `DefState.cs`, `ObjectState.cs`, `ObjectInstance.cs` model classes; `DesignStateIdGenerator` static class with unit-tested `DS_`/`OS_`/`OI_` ID computation and a declared `IDR_` prefix constant.

## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| User-supplied ObjectRef string → DesignStateIdGenerator hash input | ObjectRef is a free-text user-supplied string (Rhino/Speckle GUID); it is hashed, never executed or interpolated into Cypher directly by this plan's code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-------------------|
| T-07-04 | Tampering | DesignStateIdGenerator.ComputeObjectStateId / ComputeObjectInstanceId | accept | SHA256 truncated to 16 hex chars is a content-addressable identifier, not a security boundary; collision risk is negligible (documented in PITFALLS.md P5) and consistent with the existing v2.0 DefState ID scheme |
| T-07-05 | Repudiation | ObjectInstance.CreatedAtUtc / ObjectState.CapturedAtUtc | accept | Timestamps are local DateTimeOffset.UtcNow, informational only — no audit-log requirement exists for this phase's scope |
| T-07-06 | Information Disclosure | ObjectRef (Rhino/Speckle GUID) embedded in SHA256 hash input | accept | The GUID itself is not derivable from the truncated hash output (one-way function); no new disclosure surface beyond what the existing DS_ scheme already exposes |
</threat_model>
