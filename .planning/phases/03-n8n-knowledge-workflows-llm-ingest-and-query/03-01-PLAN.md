<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser -> n8n webhook | Untrusted NL text enters as `prompt_text` |
| n8n -> Ollama | Prompt text forwarded to LLM for processing |
| n8n -> Neo4j | Cypher statements with user-influenced parameters |
| n8n -> data-service | Execution result payload posted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Tampering | n8n Build Cypher node | mitigate | Hardcode `graph: 'KnowledgeGraph'` in Function node parameters — never read graph label from user input. Prevents cross-graph writes. |
| T-3-02 | Injection | n8n Neo4j HTTP calls | mitigate | All Cypher uses parameterized queries ($noteId, $project, $query) — prompt_text is NEVER interpolated into Cypher strings. Validated in acceptance criteria. |
| T-3-03 | Information Disclosure | Ollama prompt | accept | prompt_text is sent to local Ollama instance (same Docker network). No external API call. Low risk — no PII in architectural notes. |
| T-3-04 | Denial of Service | Ollama inference timeout | accept | 900s timeout matches existing workflows. Ollama runs locally. No amplification vector. |
| T-3-05 | Spoofing | n8n webhook endpoint | accept | Webhooks are internal (Docker network + Nginx proxy). No authentication on existing webhooks either. Consistent with current threat model. |
</threat_model>
