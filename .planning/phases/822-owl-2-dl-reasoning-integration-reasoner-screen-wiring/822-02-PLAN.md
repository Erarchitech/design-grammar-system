<objective>
Make the `data-service` reasoning proxy correct for criterion 3 and flip HermiT to a live engine for
D-04.

Two data-service concerns:
1. **Proxy timeout reconciliation (RESEARCH Pitfall 2).** `/reasoner/consistency` currently uses a
   5s read timeout (app.py L1186) while the sidecar's own hard timeout defaults to 90s. As a
   project's rule corpus grows, a genuine slow-but-valid HermiT run would be masked as a data-service
   transport timeout long before the sidecar's honest semantic timeout (D-08) could fire. Bump the
   read timeout to track the sidecar ceiling with margin; keep connect short so an unreachable
   sidecar still fails fast (non-blocking).
2. **HermiT registry status (D-04).** Flip `reasoner.py`'s HermiT entry from `placeholder` to
   `integrated` so it becomes the one live engine — the existing "Integration pending" badge (driven
   by `status === "placeholder"`) drops for HermiT while Pellet keeps it. This is the registry's
   designed extension point (reasoner.py module docstring).

Purpose: Guarantee a hung/slow reasoner surfaces as a distinct, honest outcome (never a silent
pass/fail, never a synchronous block) and expose HermiT as runnable to Plan 03.
Output: Reconciled proxy timeout + integrated HermiT status + backend tests.
</objective>
