    DECONSTRUCT templates (used by Plan 2's DESIGN STATE DECONSTRUCT and OBJECT DECONSTRUCT -- per D-07 Warning level):

       "DESIGN STATE DECONSTRUCT: DesignState input is required. Connect a DESIGN STATE or VALIDATION GRAPH component to the DesignState input."

    2. `DesignStateDeconstructCastFailed()` returns a string:
       "DESIGN STATE DECONSTRUCT: Could not unwrap the DesignState input. Ensure the input is connected to a DESIGN STATE or VALIDATION GRAPH component."

    3. `ObjectDeconstructInputMissing()` returns a string:
       "OBJECT DECONSTRUCT: ObjState input is required. Connect a DESIGN STATE DECONSTRUCT or OBJECT STATE component to the ObjState input."

    4. `ObjectDeconstructCastFailed()` returns a string:
       "OBJECT DECONSTRUCT: Could not unwrap the ObjState input. Ensure the input is connected to a DESIGN STATE DECONSTRUCT or OBJECT STATE component."

    REINSTATE templates (used by Plan 3's PARAMETER REINSTATE -- per D-02 Target Required, D-01 wire discovery):

    5. `ReinstateTargetRequired()` returns a string:
       "PARAMETER REINSTATE: Target input is required. Wire the 'State' output of a PARAMETER STATE component to the Target input."

    6. `ReinstateSourceNotFound()` returns a string:
       "PARAMETER REINSTATE: Could not find a PARAMETER STATE component upstream of the Target input. Ensure the Target input is wired to a PARAMETER STATE component's State output."

    FormatStatus and FormatMessage (moved from old ReinstateComponent into this shared Core class so DG.Tests can test them; Plan 3 calls these instead of inlining):

    7. `FormatStatus(ReinstatementResult result)` returns string:
       - result.Applied and result.AppliedCount > 0: $"Applied {result.AppliedCount} parameters"
       - result.Aborted and result.BlockedCount > 0: $"Aborted: {result.BlockedCount} blocked"
       - result.UnchangedCount > 0: "Unchanged (same state)"
       - otherwise: "Idle"

    8. `FormatMessage(ReinstatementResult result)` returns string:
       - result.Applied and result.AppliedCount > 0: $"Applied {result.AppliedCount}"
       - result.Aborted and result.BlockedCount > 0: "Aborted"
       - result.UnchangedCount > 0: "Unchanged"
       - otherwise: "Idle"

    Follow the existing pattern: all methods are `public static string`. The templates use the What+Where+How-to-fix pattern (component name + what's wrong + what to do). The FormatStatus/FormatMessage methods take ReinstatementResult (DG.Core.Models namespace).

Verify the string is non-empty and ends with a period.

    - For FormatStatus: 4 Facts covering each branch (Applied with count, Aborted with count, Unchanged >0, Idle). Each creates a ReinstatementResult with appropriate values and verifies the output string matches the expected format.

    - For FormatMessage: 4 Facts covering each branch (Applied, Aborted, Unchanged, Idle), same pattern.

    - An `AllMethods_ReturnNonEmpty` aggregate test (following existing pattern) that calls every new method and Assert.NotEmpty.

    Follow the existing test conventions: `Assert.Contains` for keyword verification, `Assert.EndsWith(".")` for sentence format, inline array/collection construction for ReinstatementResult test instances.
