  <action>
Create `DG/src/DG.Grasshopper/Components/ReinstateComponent.cs` wrapped in `#if GRASSHOPPER_SDK`.

**Component metadata:**
- Name: "REINSTATE", NickName: "REINSTATE"
- Description: "Apply a saved design state back to Grasshopper parameters. Requires a DESIGN STATE component reference for target resolution."
- Category/Subcategory: `DgComponentCategory.Category` / `DgComponentCategory.Subcategory`
- New GUID (generate a stable one)

**Inputs (RegisterInputParams):**
1. `State` (Generic, item) — `DesignStateSnapshot` to apply. Wire from VALIDATION RUNS "States" output or directly from DESIGN STATE.
2. `DesignState` (Generic, item) — Reference to the DESIGN STATE component instance. Used to resolve write targets by walking its input ports' Sources.
3. `Apply` (Boolean, item, default: false) — Rising-edge trigger. Component fires on false→true transition only.

**Outputs (RegisterOutputParams):**
1. `Result` (Generic, item) — Full `ReinstatementResult` object for downstream tooling.
2. `Report` (Text, list) — Per-parameter status lines. Format: "{ParameterId}: {Status} — {Detail}". One line per snapshot parameter.
3. `Status` (Text, item) — Summary message: "Applied N parameters" / "Aborted: M blocked" / "Unchanged (same state)" / "Idle".

**Instance state (per D-10, not persisted):**
- `private bool _lastApplyInput = false;` — for rising-edge detection
- `private string? _lastAppliedStateId = null;` — StateId guard
- `private ReinstatementResult? _latestResult = null;` — cache for output

**SolveInstance algorithm:**

1. **Read inputs:** Unwrap State (DesignStateSnapshot), DesignState (DesignStateComponent), Apply (bool). If State or DesignState null/missing, set error and return.

2. **Rising-edge detection (D-08):**
   ```csharp
   var currentApply = applyInput; // the bool read from input
   var isRisingEdge = currentApply && !_lastApplyInput;
   _lastApplyInput = currentApply;
   if (!isRisingEdge)
   {
       // Output cached result if available, otherwise "Idle"
       SetOutputs(da, _latestResult, "Idle");
       return;
   }
   ```
   This ensures sustained `true` doesn't re-fire. A GH Button emits a single `true` pulse which works naturally.

3. **Resolve targets (D-01 through D-04):**
   Walk the `DesignState` component's `Params.Input` list. For each input port at index `i`:
   - Derive `parameterId` from `ghParam.NickName.Trim()` (same logic as DesignStateComponent)
   - Check `ghParam.Sources` (the list of upstream IGH_Param connected to this input):
     - If `Sources.Count == 0` → `TargetResolutionStatus.Missing`
     - If `Sources.Count > 1` → `TargetResolutionStatus.Ambiguous`
     - If `Sources.Count == 1` → `TargetResolutionStatus.Resolved`
       - Determine `TargetType` by inspecting the source param type:
         - `Param_Number` or source that outputs double/float → `DesignStateParameterType.Number`
         - `Param_Integer` or source that outputs int/long → `DesignStateParameterType.Integer`
         - `Param_Boolean` or source that outputs bool → `DesignStateParameterType.Boolean`
         - Use the source's `TypeName` or try casting to specific GH param types
       - Determine domain bounds: If the source is a `GH_NumberSlider`, read `Slider.Minimum` and `Slider.Maximum`. Otherwise `DomainMin`/`DomainMax` = null.
   - Build `ResolvedTarget` record for each.

4. **Validate via Core service:**
   ```csharp
   var service = new DesignStateReinstatementService();
   var result = service.Validate(snapshot, resolvedTargets, _lastAppliedStateId);
   ```

5. **Write values (only if result.Applied == true):**
   If `result.Applied`:
   - For each parameter in snapshot where the corresponding target was `Resolved`:
     - Get the single source IGH_Param from `DesignState.Params.Input[i].Sources[0]`
     - Write the value:
       - For `GH_NumberSlider`: set `Slider.CurrentValue` (or use `SetSliderValue` — check API)
       - For generic params: clear `PersistentData`, add the typed GH_Goo value, call `ExpireSolution(false)` on the source's owner
     - After all writes: call `ExpireSolution(true)` on the document to trigger re-solve
   - Update guard: `_lastAppliedStateId = snapshot.StateId;`

6. **Set outputs:**
   - `Result` → the `ReinstatementResult` object
   - `Report` → list of formatted strings from `result.Reports`
   - `Status` → summary string based on result flags
   - Cache: `_latestResult = result;`
   - Set `Message` property on component for canvas display (e.g., "Applied 5" / "Aborted" / "Unchanged")

**Important notes:**
- Wrap entire file in `#if GRASSHOPPER_SDK` / `#endif` per project convention.
- Use `GhCastingHelpers.Unwrap<T>()` for input unwrapping (established pattern from ValidationRunsComponent).
- For slider value writing: use `GH_NumberSlider` API. Check if `slider.SetSliderValue(value)` exists; otherwise set via `slider.Slider.Value = (decimal)value` and expire.
- For boolean toggle writing: access `GH_BooleanToggle` and set `Value` property, then expire.
- If neither slider nor toggle, attempt generic `PersistentData` write pattern.
