## Data shape consumed (from Plan 823-03)
The selected-run `view` payload carries `shaclReport`:
- `undefined`/`null` → State 2 (not checked — pre-823 run)
- `{ status: "unavailable" | "timeout" }` → State 3 (not evaluated)
- `{ results: [] }` (status ok, empty) → State 1 (conforms)
- `{ results: [ { severity, what, where, howToFix, focusLabel, shapeId } ] }` → State 4 (findings)

**Hard rule (UI-SPEC):** the UI keys its states on `results.length`, NEVER on a `conforms` boolean — allow_infos/allow_warnings means `conforms` stays true when only info/warning findings exist, so a conforms flag would conflate warnings with conformance. State 1 renders only when `results.length === 0`. `shapeId` is present in the data but MUST NOT be rendered anywhere.
