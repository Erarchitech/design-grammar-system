<objective>
Surface the SHACL report on the Grasshopper side (D-15): add `ErrorMessageTemplates.ShaclViolation(...)` in DG.Core (What+Where+How-to-fix house style), extend the `ValidationPublishResponse` C# DTO to deserialize the publish response's `shacl` block, and have `ValidatorComponent` render findings into its Report output + runtime messages — capped at Warning so a data-integrity finding never makes the component look failed/errored.

Purpose: Requirement SHCL-02 requires SHACL violations to surface through DG's existing ErrorMessageTemplates in the architect's plain-language house style, on the same VALIDATOR component that publishes the run.

Output: a new template method + its tests, two new DTO classes + a round-trip test, and the ValidatorComponent surfacing block.

## Contract consumed (from Plan 823-02 / 823-03 — camelCase, exact)
The publish response carries top-level `shacl` = `{ status, conforms, results, counts }` where each `results` entry is `{ severity, what, where, howToFix, focusLabel, shapeId }`. System.Text.Json is configured with `JsonNamingPolicy.CamelCase` and silently nulls unmatched fields — the C# property names MUST camelCase-match these keys exactly or the DTO deserializes to empty values without throwing (RESEARCH Pitfall 4).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| data-service HTTP body → C# DTO | untrusted JSON deserialized by System.Text.Json (silently nulls unmatched fields) |
| SHACL findings → architect | findings rendered in Report/runtime messages must stay plain-language (already stripped upstream) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-823-05 | Tampering | ValidationPublishResponse deserialization | high | mitigate | Exact camelCase field names + a round-trip test asserting every field populated — prevents silent-null data loss (System.Text.Json does not throw on mismatch) |
| T-823-10 | Denial of Service | GH component erroring on data findings | medium | mitigate | Severity cap at Warning/Remark (D-15) so a SHACL finding never fails the component or blocks the canvas |
