<objective>
Wire the SHACL validation into `data-service`'s publish path as a non-fatal sidecar proxy (D-01/D-02), persist the report as `shaclReportJson` on the `ValidationRun` node and return it in the publish response (D-06), and extend the runs/view fetch surface so the Model screen can read a parsed `shaclReport` (D-17).

Purpose: This is the integration seam that makes SHACL run "on every validation run" without ever endangering the Speckle-publish hot path. It stores and surfaces the canonical envelope produced by Plan 823-02.

Output: `_call_shacl_validate` + `_persist_shacl_report` in `data-service/app.py`, wiring inside `publish_validation`, view/list parsing, and a new `test_shacl_proxy.py`.

## Envelope handling (consumes Plan 823-02's canonical body)
- dg-reasoner `/shacl/validate` success body = `{conforms, results, counts}`. The proxy wraps it as `{"status": "ok", "conforms", "results", "counts"}`.
- On unreachable → `{"status": "unavailable"}`; on timeout (httpx timeout OR body `error == "timeout"` OR HTTP 504) → `{"status": "timeout"}`; any other exception → `{"status": "unavailable"}`. The proxy NEVER raises.
- Stored `shaclReportJson` = `json.dumps(<the wrapped block>)`. Publish response embeds it under top-level key `shacl`. The runs/view payload embeds the parsed object under key `shaclReport`.
- Pre-823 runs have no `shaclReportJson` property → view/list emit `shaclReport: null` (or omit the key) — never a fabricated error.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VALIDATOR/browser → data-service | publish request carries project + run_id |
| data-service → dg-reasoner | fixed internal URL from `DG_REASONER_URL` env; no user-controlled URL parts |
| data-service → Neo4j | shaclReportJson persisted + read via parameterized Cypher |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-823-01 | Tampering | _persist_shacl_report Cypher | high | mitigate | Parameterized `$project`/`$runId`/`$shaclReportJson` — never string-interpolate into Cypher (acceptance-checked) |
| T-823-03 | Spoofing/SSRF | dg-reasoner proxy call | low | accept | Target URL is the fixed internal `DG_REASONER_URL` env value; no part of the URL is user-controlled — no SSRF surface |
| T-823-08 | Denial of Service | SHACL call holding the publish hot path open | medium | mitigate | Short `DG_SHACL_HTTP_TIMEOUT_SECONDS` (default 15s) + non-fatal catch-all so a slow/hung sidecar degrades to `unavailable`/`timeout`, never blocks publish |
| T-823-02 | Information Disclosure | shaclReport passed through to UI | high | mitigate | The report was already stripped of raw RDF in Plan 823-02; data-service forwards it verbatim without re-introducing IRIs |
| T-823-SC | Tampering | package installs | low | accept | No new packages introduced |
</threat_model>
