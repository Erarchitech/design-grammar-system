# Plan 814-01: Reasoner Screen

## Objective

The Reasoner region lets the user select the reasoner model for the Validation Graph. HermiT and Pellet ship as clearly-labeled placeholders ("integration pending" — selection is persisted server-side but does not yet drive validation). Selection survives revisits.

## Context (read before coding)

- `data-service/app.py` + `data-service/connectors.py` — persistence and endpoint conventions established in Phase 812 (JSON store under `data-service/data/`, structured errors). Reasoner settings need no encryption (no secrets).
- `ui-v2/src/screens/ReasonerScreen.jsx` — skeleton from Phase 810; fill it in.
- `ui-v2/src/lib/` conventions per `modelApi.js`; nginx `/data-service/` proxy.
- Design system as in sibling plans; no new shared primitives.

## Tasks

### Task 1: backend — `data-service/reasoner.py` + endpoints

- Registry: `[{id: "hermit", name: "HermiT", description: <one-liner: OWL 2 DL hypertableau reasoner>, status: "placeholder"}, {id: "pellet", name: "Pellet", description: <one-liner: OWL 2 DL tableau reasoner>, status: "placeholder"}]` — structured so real integrated reasoners can later carry `status: "integrated"`.
- `GET /reasoner/settings` — registry + currently selected reasoner id (null if never set); `PUT /reasoner/settings` — body `{reasoner}`, 422/400 on unknown id; persisted to `data/reasoner-settings.json`.
- `data-service/tests/test_reasoner.py`: registry shape, select/persist round-trip, unknown id rejected.

### Task 2: ReasonerScreen UI

- `ui-v2/src/lib/reasonerApi.js`: `getReasonerSettings()`, `selectReasoner(id)`.
- Two selectable reasoner cards (HermiT, Pellet) with description; selected card carries the active/selection treatment (REAS-01)
- Selection persists via PUT and is shown active on remount/revisit (REAS-02)
- Prominent "integration pending" annotation on placeholder entries + a screen-level note that selection does not yet change validation behavior (REAS-03)

`python -m pytest data-service/tests/ -q` green; `npm --prefix ui-v2 run build` passes; live round-trip if data-service reachable, else note it.
