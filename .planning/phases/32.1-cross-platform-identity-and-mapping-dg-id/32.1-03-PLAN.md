<objective>
Build the data-service identity registry API core: `data-service/dg_identity.py` plus the `/identity/mint`, `/identity/resolve`, `/identity/bind`, and GET+DELETE `/identity/{dgId}/representations` routes. This is the single source of truth every current and future DG connector speaks for minting persistence, native-id ↔ dgId resolution, and platform-representation binding/detaching — over parameterized, project-isolated Cypher with structured errors that never silently misbind.

Purpose: DGID-02 (registry binds each dgId to per-platform representations with connector provenance; representations attach AND detach without changing the dgId), DGID-03 (counterpart objects across platforms resolve to one identity), DGID-05 (mint/resolve/bind over parameterized project-isolated Cypher with What+Where+How-to-fix errors, never silent misbinding).
Output: the identity module, its routes and error factories, and a test suite proving the round-trips and the anti-misbinding guard.
</objective>

|----------|-------------|
| client → /identity/* API | Untrusted native-id / dgId / project strings cross into parameterized Cypher against the shared single Neo4j database. |
| connector → registry | A connector asserts a native-id ↔ dgId binding; a wrong or forged assertion could repoint identity. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-32.1-03b | Tampering / Repudiation | silent misbinding (native id repointed to a different dgId) | high | mitigate | `bind_representation` MATCHes any existing `Representation {nativeId, platform, project}` and raises `DGID_AMBIGUOUS_BINDING` (409) on a dgId mismatch — never a bare idempotent MERGE that would silently reassign (Pitfall 4).
