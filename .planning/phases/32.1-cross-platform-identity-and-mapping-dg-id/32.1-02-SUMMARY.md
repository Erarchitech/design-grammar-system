# Phase 32.1 Plan 02: Cross-Platform Identity Documentation Summary

Wrote the two documentation deliverables that turn the DG ID scheme into a referenceable contract: `spec/DG-ID.md` (the normative identity spec) and the DGID-06 ADR positioning the scheme against Rhino.Inside.Revit, Speckle, IFC GlobalId, Revit UniqueId, and BHoM.

## What was built

## Key design decisions (documented, not newly made — locked upstream in CONTEXT/RESEARCH)

- The hash-input contract `project|definitionId|cgId` is pinned as the one contract both the C# and Python minters follow, guarded by a shared golden vector — the cross-language parity anchor.
- Revit binds `UniqueId`, never `ElementId` — the load-bearing survey distinction, stated in both documents with the stability rationale.
- `Representation`/`SharedProperty` under `graph:'Computgraph'` — no fifth partition, minimizing schema-propagation surface.
- Rename re-mints by default; the escape hatch is a documented future seam only.
