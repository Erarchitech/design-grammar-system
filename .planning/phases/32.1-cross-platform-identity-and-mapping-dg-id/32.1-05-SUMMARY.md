# Phase 32.1 Plan 05: Cross-Platform Identity — Wire DG ID into Computgraph Serialization

Wires the deterministic DG identity (`dg:XXXXXXXXXXXXXXXX`) into Phase 32's Computgraph serialization surface: every tagged ontological entity (Object, Procedure, Pattern, Parameter, Interface) now carries an additive nullable `DgId` field in-memory and round-trips it as a camelCase `"dgId"` key in the `cgContextJson v1` envelope — without a version bump and without breaking any pre-32.1 payload reader.

## Files Created

**`DG/src/DG.Core/Services/CgContextDgIdAssigner.cs`** — GH-free static service that walks a parsed `CgContext` and stamps a deterministic `DgId` onto every tagged entity. Uses `DgIdMintingService.Mint(project, definition.DocumentId, entityKey)` where:
- Object: `entityKey = "obj:" + Name` (CgObject has no Id)
- Procedure/Pattern/Parameter/Interface: `entityKey = entity.Id`

**`DG/tests/DG.Tests/Identity/CgContextDgIdAssignerTests.cs`** — 5 facts proving:
1. All tagged entities get non-null `dg:`-prefixed DgId
2. Two identical contexts produce byte-identical per-entity DgIds (determinism)
3. Object DgId derives from Name with `"obj:"` prefix
4. Typed-entity DgIds derive from their Id
5. Different project changes every assigned DgId

## Files Modified

**5 Computgraph entity models** (`CgObject.cs`, `CgProcedure.cs`, `CgPattern.cs`, `CgParameter.cs`, `CgInterface.cs`):
- Each gains `public string? DgId { get; set; }` (nullable, defaults null — "not yet minted")
- Strictly additive: no existing member changed type or name

**`ComputgraphContextSerializer.cs`**:
- Private DTOs (`CgObjectDto`, `CgProcedureDto`, `CgPatternDto`, `CgParameterDto`, `CgInterfaceDto`) each gain `public string? DgId { get; init; }`
- ToDto and FromDto methods map DgId through for all 5 entity types
- `schemaVersion` stays `"cg-context-1"` — additive field, no version bump
- camelCase Options automatically render `"dgId"` key in JSON

**`ComputgraphContextSerializerTests.cs`** — 4 new facts added:
1. Serialize with DgIds emits `"dgId":"dg:` key in JSON
2. Round-trip preserves per-entity DgId
3. Idempotency holds with DgIds present (`Serialize(Deserialize(s)) == s`)
4. Pre-32.1 payload (no `dgId` keys) deserializes with null DgId on every entity

Documented here so Plan 06 can confirm the field and serializer support already exist.

## Cross-Phase Contract (locked for Phase 36)

The entityKey derivation is contractually locked:
- **DefinitionId** = `context.Definition.DocumentId` (the stable document GUID)
- **Object entityKey** = `"obj:" + context.Object.Name` (since CgObject carries `Name`, not `Id`)
- **Typed entity entityKey** = `entity.Id` (Procedure/Pattern/Parameter/Interface)

Phase 36's publish path must mint / carry dgIds with the *identical* inputs (`project`, `definition.DocumentId`, `entityKey`) so a pre-minted (assigner) dgId equals an API-minted (`/identity/mint`) dgId for the same entity.
