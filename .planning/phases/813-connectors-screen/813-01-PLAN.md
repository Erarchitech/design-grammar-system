# Plan 813-01: Connectors Screen

## Objective

The Connectors region lists all 14 connectors grouped in their 5 categories, lets the user mint a credential (token shown once, copyable for pasting into the connector component in the target software), shows per-connector activation status + last-connection date, and supports revocation.

## Context (read before coding)

- Backend shipped in Phase 812 — read `data-service/connectors.py` and the connector endpoints in `data-service/app.py` for exact shapes: `GET /connectors` (registry + status `never_connected|active|stale` + `last_connection` + credential summaries), `POST /connectors/{id}/credentials` (returns `{credential_id, token}` once, token prefix `dgc_`), `DELETE /connectors/{id}/credentials/{credential_id}`, `POST /connectors/heartbeat` (Bearer token — used by connector plugins, not by this screen).
- Reach the API the same way existing screens reach data-service: read `ui-v2/src/lib/modelApi.js` for the base-path convention (nginx `/data-service/` proxy in prod; check `ui-v2/vite.config.js` dev proxy).
- `ui-v2/src/screens/ConnectorsScreen.jsx` — skeleton from Phase 810; fill it in.
- Design system: `dg-frost` cards, `dg-annotation` eyebrows, pill geometry; Signal Red is selection/fail-only — use the status token conventions (`--status-*` in `ui-v2/src/styles/tokens/`) for status dots. No new shared primitives in `components/`.

## Tasks

### Task 1: `ui-v2/src/lib/connectorsApi.js`

`listConnectors()`, `createCredential(connectorId, label)`, `revokeCredential(connectorId, credentialId)`.

### Task 2: ConnectorsScreen UI

- Five category sections in brief order (VPL Platforms, BIM Authoring, BIM Coordination, BCF Trackers, Visualization), 13 connector cards with name + status dot + status text + last-connection date (formatted, or "never connected") (CONN-01, CONN-03)
- Per connector: expandable credentials area — list existing credentials (label, created date, revoked flag), "Create credential" with optional label → one-time token reveal panel with a Copy button (clipboard API) and an explicit "shown once — paste into the connector component in <software>" annotation (CONN-02)
- Revoke action per credential with confirm affordance; list refreshes after revoke (CONN-04)
- Refresh control re-fetching statuses; loading/error states

### Task 3: Verify

`npm --prefix ui-v2 run build` passes. If data-service is reachable, run a live create→copy→revoke pass against a test connector and note results; otherwise verify shapes against backend code and say so in the SUMMARY.
