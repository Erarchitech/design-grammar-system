  <name>Task 1: Thinking-core video composite with procedural fallback + start/stop API in graphEngine</name>
  <files>ui-v2/src/graph/graphEngine.js</files>
  <action>
Add a "thinking core" subsystem to GraphEngine (requirement 1 + 3).

State (constructor): initialize `this._think = 0` (current intensity 0..1), `this._thinkTarget = 0`, `this._thinkVideo = null`, `this._thinkVideoReady = false`, `this._thinkVideoFailed = false`.

Video setup: add a private method `_initThinkVideo()` and call it once from `start()` (after `resize()`), guarded so it only creates the element once. Create an offscreen `HTMLVideoElement` via `document.createElement("video")` (NOT appended to the DOM), set `muted = true`, `loop = true`, `playsInline = true`, `preload = "auto"`, and `src = "/dg-think-sphere.mp4"`. On `"canplay"` (or `"loadeddata"`) set `_thinkVideoReady = true`. On `"error"` and `"stalled"` set `_thinkVideoFailed = true`. Store on `this._thinkVideo`. Do NOT assume the file exists — a missing/404 asset must set `_thinkVideoFailed` and never throw.

Public API:
- `startThinking()` — sets `this._thinkTarget = 1`; if `_thinkVideo` exists and not failed, call `_thinkVideo.play()` and swallow the returned promise rejection (muted autoplay after a click gesture is allowed; a rejection must not surface).
- `stopThinking()` — sets `this._thinkTarget = 0`. (Do not pause the video here; let it pause when intensity reaches ~0 inside the update step, so the fade-out still shows frames.)

Intensity ramp: add `updateThinking(dt)` and call it from `tick(ts)` right AFTER the camera lerp and BEFORE `this.drawGraph()`. Compute an EFFECTIVE target: `const target = (this._streams && this._streams.length) ? Math.max(this._thinkTarget, 0.6) : this._thinkTarget;` — while streams are alive the sphere holds at least 0.6 intensity even after stopThinking(), so streams visually pour out of a still-glowing sphere and the sphere only fully fades after the LAST stream lands (continuity requirement). Ramp `_think` toward `target` (approach rate ~`Math.min(1, dt*4)` so fade-in/out ≈0.25-0.4s). When `_think` drops below ~0.01 and effective target is 0, snap to 0 and pause the video if it is playing. Also call the Task-2 stream update from here (Task 2 will add `updateStreams(dt)` — call it BEFORE the effective-target computation so stream expiry is reflected the same frame).

Loop note: /dg-think-sphere.mp4 is a pre-baked SEAMLESS loop (4s, tail crossfaded into head). `loop = true` is sufficient — do NOT add ping-pong, restart-crossfade, or seek logic.

Rendering: add `drawThinkingCore(cx, cy, base, t)` and call it near the END of `drawGraph()` (after nodes/spire), passing the already-computed `cx, cy, base` and `this._lt || 0`. Behavior:
- If `this._think < 0.005` return immediately.
- Core radius `R = base * 0.30`.
- Paint a local dark radial backdrop first: `ctx.createRadialGradient(cx,cy,0, cx,cy,R)` from ~`rgba(8,8,12, 0.92*_think)` at center fading to `rgba(8,8,12,0)` at R, filled over an arc of radius R (normal source-over). This is the "pure black background" the wisps read against.
- Then composite the wisps with `ctx.save(); ctx.globalCompositeOperation = "lighter"; ctx.globalAlpha = _think;`
  - If `_thinkVideoReady && !_thinkVideoFailed`: `ctx.drawImage(this._thinkVideo, cx - R, cy - R, 2*R, 2*R)` (720x720 square → the black corners add nothing under `lighter`, so it reads as a sphere).
  - Else (fallback, requirement/constraint: graceful degrade): draw a procedural glow — a radial gradient of near-white (`rgba(245,245,248, a)`) with a gentle pulse `a = _think * (0.55 + 0.25*Math.sin(t*2.2))`, plus a couple of concentric faint rim strokes — so a working core sphere shows even with no video.
  - `ctx.restore()` and reset `globalAlpha = 1` / `globalCompositeOperation = "source-over"`.

Do NOT change existing draw ordering of guides/field/edges/nodes. The dark disc briefly dimming central-orbit nodes during processing is intended.
  </action>

<task type="auto">
  <name>Task 2: Core-to-node streams + per-node emergence system in graphEngine</name>
  <files>ui-v2/src/graph/graphEngine.js</files>
  <action>
Add the outward-stream + node-emergence subsystem (requirement 2 + 4). All monochrome ink, matching the existing datascape aesthetic (reuse `TH.ink`, `gauss(R)`, `rng(seed)` already in the file; do NOT introduce new colors).

State (constructor): `this._streams = []`, `this._emerge = Object.create(null)` (map neoId → emergence 0..1). In `setData()`, reset both (`this._streams = []; this._emerge = Object.create(null);`) so a plain reload/restore never carries stale streams.

Public API:
- `snapshotNodeIds()` — return a `Set` of every `nd.neoId` across `this.rings`. GraphScreen calls this BEFORE a mutating turn.
- `emitNewNodeStreams(prevIdSet)` — scan `this.rings`; for each node whose `nd.neoId` is not in `prevIdSet`, create a stream targeting that node and seed its emergence at 0 (`this._emerge[nd.neoId] = 0`). Cap total emitted streams at ~80 (still seed emergence for any beyond the cap so those nodes still pop in, just without a filament). If `prevIdSet` is empty (first ingest into an empty graph) this legitimately streams to all nodes — that is the intended dramatic first-load.

Stream object shape: `{ g, i, neoId, p: 0, life: 0.85 + Math.random()*0.5, seed }`. Precompute ~18-26 particle descriptors per stream using `rng(seed)`: each `{ u: base position along path 0..1, off: gauss perpendicular amplitude, ph: phase, s: 0.6-1.6 size, al: 0.12-0.5 alpha }`.

Update: add `updateStreams(dt)` (called from `updateThinking(dt)` per Task 1). For each stream advance `stream.p += dt / stream.life`. Drive emergence from head progress: `this._emerge[neoId] = clamp((stream.p - 0.6) / 0.4, 0, 1)`. When `stream.p >= 1.25` (head arrived + short tail retract) remove the stream and `delete this._emerge[neoId]` (absent key = fully materialized node). Keep `_streams` filtered to live entries.

Render: add `drawStreams(cx, cy, base, t)` and call it from `drawThinkingCore` AFTER the core sphere is painted (streams overlay, radiating from the sphere) — pass the same `cx,cy,base,t`. Gate the whole thing on `this._streams.length`. For each stream:
- Compute the live target with the SAME math as `posOf`: `nd = this.rings[g].nodes[i]; a = nd.ang + this._rot[g][nd.orbit]; R = this._orbR[g][nd.orbit]*base; tx = cx + cos(a)*R; ty = cy + sin(a)*R`. (Guard: skip if `this.rings[g]` or the node is missing — rings can change between frames.)
- Direction unit `(dx,dy)` from core `(cx,cy)` to `(tx,ty)`, length `len`; perpendicular `(-dy,dx)/len`.
- CONTINUITY (locked): streams must originate at the SPHERE'S RIM, not the exact center — offset the stream origin to `ox = cx + dx/len * base*0.24; oy = cy + dy/len * base*0.24` (just inside the core radius `base*0.30`) and lay particles along `origin→target` from there. Combined with the Task-1 hold-at-0.6-intensity-while-streams-alive rule, the streams read as pouring continuously out of the still-glowing sphere.
- Head reveal: only draw a particle when `stream.p*1.12 >= part.u`; fade each particle in near the moving head and out as the tail retracts once `p>1`. Particle position = `origin + dir*len*part.u + perp * part.off * Math.sin(part.ph + t*3 + part.u*6) * (1 - part.u)*coreScale`. Draw as small ink rects with `ctx.fillStyle = TH.ink` and per-particle `globalAlpha` (same technique as `drawField`).
- Draw a slightly larger/darker ink head dot at `origin + dir*len*min(1,stream.p)`.
- Restore `globalAlpha = 1` at the end.

Node emergence in draw: in BOTH node-draw loops in `drawGraph()` (active-layer ~L716-742 and dimmed inactive-layer ~L640-653), look up `const emv = this._emerge[nd.neoId]; const emK = emv == null ? 1 : emv;` and apply `radius *= (0.25 + 0.75*emK)` and multiply the node's `globalAlpha` by `emK` (combine with the existing `faded` alpha in the active loop). Absent key ⇒ emK=1 ⇒ unchanged rendering for all pre-existing nodes.
  </action>
