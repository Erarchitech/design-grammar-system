    **Per D-06:** Create `DG/src/DG.Core/Services/ErrorMessageTemplates.cs` as a `public static class` with methods:

    1. `SerializationFailed(string parameterId, SerializationError error)` — maps each enum value to a What+Where+Fix string. Examples:
       - `NoStateProvided` → `"State capture failed: no state provided for parameter '{parameterId}'. Connect a DESIGN STATE component upstream."`
       - `MalformedStatePayload` → `"State serialization failed: parameter '{parameterId}' produced invalid payload. Disconnect and reconnect the parameter input."`
       - `MissingParameterId` → `"State capture incomplete: parameter '{parameterId}' has no identifier. Right-click the input and set a NickName."`
       - `TimestampMissing` → `"State capture failed: parameter '{parameterId}' has no timestamp. Re-capture the design state."`

    2. `ReinstatementBlocked(string parameterId, ReinstatementStatus status, string detail)` — wraps existing reinstatement error detail in D-06 template. Format: `"Reinstatement blocked: parameter '{parameterId}' {detail}. {fix based on status}"`. Fix hints:
       - `MissingTarget` → `"Reconnect the original slider or toggle."`
       - `TypeMismatch` → `"Reconnect a matching slider or recapture state."`
       - `AmbiguousTarget` → `"Ensure only one slider has this NickName."`
       - `OutOfRange` → `"Adjust slider domain to include the saved value."`
       - Default → `"Check the parameter connection."`

    3. `ValidationInputMissing(string inputName)` — `"Validation input missing: {inputName} is required. Connect {inputName} to the component input."`

    4. `PublishFailed(string project, string errorDetail)` — `"Validation publish failed for project '{project}': {errorDetail}. Check data-service connection and Speckle configuration."`

    5. `DesignStateTypeUnsupported(string parameterName, string typeName)` — `"Design state capture skipped: parameter '{parameterName}' has unsupported type '{typeName}'. Only Number, Integer, and Boolean inputs are supported."`
