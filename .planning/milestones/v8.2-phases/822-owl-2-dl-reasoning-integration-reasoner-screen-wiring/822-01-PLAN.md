<objective>
Enrich the `dg-reasoner` sidecar's `/reason/consistency` response so each unsatisfiable class is
returned as `{iri, label}` instead of a bare IRI string, resolving D-13 (RESOLVED = option a,
sidecar enriches labels) and enabling D-02 (Inconsistent verdict lists classes by human label,
never a raw IRI).

This is an explicit **upstream Phase-821-touch**: `dg-reasoner/reasoning.py` was shipped by 821 but
821 is not formally closed. The change is small and contained — the label data is already loaded
in-memory in the same reasoning call (via `ontology_export.build_graph`'s `rdfs:label` triples,
ontology_export.py L234-237), so enrichment costs one attribute read per class and zero new Neo4j
queries.

Purpose: Give the frontend (Plan 03) a ready-to-render human label so it never surfaces raw RDF
vocabulary to the architect (D-02, criterion 2).
Output: Updated `unsatisfiable_classes` element shape + regression tests.
</objective>
