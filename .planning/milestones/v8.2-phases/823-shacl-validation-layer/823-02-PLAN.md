## CANONICAL shaclReport ENVELOPE (pin — identical field spelling in every layer, camelCase)

dg-reasoner `/shacl/validate` **success body** returns exactly:
- `conforms`: boolean | null — derived as `not any(r.severity == "violation")`; `null` only on timeout
- `results`: array of finding objects (empty array == conforms). **Array key is `results`** (preserves the 821 backward-compat contract; do NOT rename to `findings`)
- `counts`: `{ "violation": <int>, "warning": <int>, "info": <int> }`

Each finding object has exactly these keys (D-10):
- `severity`: `"violation" | "warning" | "info"`
- `what`: plain-language description (from the shape's `sh:message`)
- `where`: human location string (the enriched focusLabel, plus the property path local-name when a `sh:resultPath` is present)
- `howToFix`: remediation guidance (from the shape's `dgsh:howToFix` annotation; fallback to a generic per-severity hint if the shape has none)
- `focusLabel`: human label of the focus node (rdfs:label from the data graph, else `_local_name()` fallback)
- `shapeId`: local-name of the source shape (internal identifier — downstream MUST NOT render it in the UI)

On timeout, keep the existing 821 shape: `{ "conforms": null, "error": "timeout", "timeout_seconds": N }`. The data-service proxy (Plan 823-03) wraps the success body with a top-level `status: "ok"`.

**Namespaces:** shapes file declares `dgsh: <http://example.org/design-grammar/shapes#>` (its own shape IRIs + the `dgsh:howToFix` annotation predicate) and targets `dgv:` (from Plan 823-01) and the existing `swrl:`/`ex:` Metagraph vocabulary.
