<objective>
Build the ValidGraph→RDF ABox exporter in dg-reasoner (D-04) — a new `valid_graph_export.py` sibling to `ontology_export.py` that reads the `statePayloadJson` v2 envelope stored on a `ValidationRun` Neo4j node and mints `dgv:ObjState`/`dgv:ParamState`/`dgv:PropState`/`dgv:Run` named individuals, plus the `owl:AllDifferent` (UNA) declaration Phase 821 deferred to this phase (D-05).

Purpose: This ABox is what Phase 823's SHACL layer validates against (unioned with the existing OntoGraph/Metagraph export). Without it there is no per-run instance data to check. It is the foundation the SHACL shapes + result mapping (Plan 823-02) build on.

Output: `dg-reasoner/valid_graph_export.py` with `build_valid_graph(session, project, run_id)` and `add_all_different(graph, individuals)`, plus fixture-backed unit tests.

**Canonical namespace (pin, reused by Plan 823-02's shapes):** `DGV = Namespace(f"{BASE}/valid#")` where `BASE = "http://example.org/design-grammar"` (identical to `ontology_export.BASE`). Bind prefix `dgv`. Individual IRIs are minted via the existing `mint(project, kind, key)` helper: state individuals use `mint(project, "state", stateId)`, the run individual uses `mint(project, "run", runId)`.
</objective>
