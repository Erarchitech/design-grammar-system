# Phase 825: CONNECTOR Token-Driven Simplification — User Acceptance Tests

Automatable checks all pass (see `825-VERIFICATION.md`): data-service 23/23 connector
tests, `dotnet build -c Release` 0/0, DG.Tests 236/236. The checks below need a live
**Rhino 8 / Grasshopper** session + a running **data-service** — DG.Grasshopper is excluded
from the headless test project.

**Prerequisites**
- `docker compose up -d` (data-service at `http://localhost:8000`).
- In the V2 UI Connectors screen → **Grasshopper** connector → **Create credential**: set a
  **Project** (defaults to the active project) and optional Label, Create, copy the `dgc_…`
  token (shown once).
- `dotnet build DG/DG.sln -c Release` → load the fresh `DG.gha` in Grasshopper.

---

## Test 1 — Two inputs only, token drives the connection (SC-1, SC-4)
1. Drop a **CONNECTOR** component. Confirm it has **exactly two inputs**: `Token` and `Go`
   (Connect). No URI/User/Password/DB/Project/DataServiceUrl.
2. Paste the `dgc_…` token into **Token** (type it directly — see Test 3). Set **Go = true**.

**Expect:** `Message` reads **`Connected · Auth OK`**; no Error/Warning bubble. The
**Project** output shows the project you bound at mint time (not typed on-canvas). The
**Database** output is a live connection object. UI Connectors screen shows the Grasshopper
connector "last connection = just now".

---

## Test 2 — Failure feedback, connection gated on auth (SC-5, ADR-825-6)
1. Replace the token with `dgc_bogus` (or revoke the real credential), **Go = true**.

   **Expect:** red **Error** `CONNECTOR: platform token rejected …`; `Message`
   **`Connect failed · Auth failed`**; **Database** output is a *not-connected*
   `ConnectionInfo` (no bolt attempt was made — the token is the only connection source).
2. Stop data-service (`docker compose stop data-service`), reconnect with any token.

   **Expect:** orange **Warning** `CONNECTOR: could not reach data-service …`; `Message`
   `Connect failed · Auth unreachable`; no hang beyond ~6 s.
3. Clear the token, **Go = true**.

   **Expect:** orange **Warning** `CONNECTOR: no platform token …`; `Message`
   **`Awaiting token`**.

---

## Test 3 — Token typeable live, never persisted (SC-6, Criterion 4)
1. **Type** a token directly into the Token input (do not wire a Panel). Set **Go = true**.

   **Expect:** it works (Test 1 outcome). The field is **usable** — no immediate scrub, no
   "internalised token cleared" remark on every solve (the Phase 824 behavior is gone).
2. Save the canvas as a `.gh`, open it in a text/hex viewer, search for `dgc_`.

   **Expect:** **no `dgc_` substring** — `NonPersistentStringParam.Write` cleared it at save.
3. Open a **pre-825 CONNECTOR canvas** (8-input, old GUID `24E78A17…`).

   **Expect:** a **missing-component placeholder** (GUID intentionally changed, ADR-825-3) —
   re-drop the new CONNECTOR and wire Token. *(This is the accepted, user-approved break.)*
