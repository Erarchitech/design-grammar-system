# Plan 825-02 — C#: two-input token-driven CONNECTOR

## Task 1 — HeartbeatResult carries the bundle (DG.Core)

- New `ConnectionBundle` record (DG.Core.Models): `Uri, User, Password, Database, Project`
  (all `string?`). Token-free by construction.
- Extend `HeartbeatResult` to `record struct HeartbeatResult(HeartbeatOutcome Outcome,
  string? Status, ConnectionBundle? Bundle = null)`. `NotAttempted` unchanged.
- `ConnectorHeartbeatClient.CheckAsync`: on 200, parse `project` + `neo4j.{uri,user,
  password,database}` into a `ConnectionBundle` alongside `status`. Tolerate a missing
  `neo4j` object (Bundle=null) so the existing 200 tests (status-only body) still pass.
  The token still flows only into the Authorization header.

## Task 2 — ConnectorComponent rewrite (DG.Grasshopper)

- **Inputs (SC-1):** register exactly two —
  `Token` (custom non-persistent `Param_String`, `Optional=false`) and
  `Connect` (bool, default false). Remove URI/User/Password/DB/Project/DataServiceUrl.
- `DataServiceUrl` becomes a private const default `http://localhost:8000` (ADR-825-4).
- **New GUID (ADR-825-3):** replace `24E78A17-4533-44E7-8931-1224A70A1B36` with a fresh
  GUID; record it in STATE decisions + release notes.
- **SolveInstance / RunConnectAsync (ADR-825-6):** heartbeat first. Build a
  `ConnectionInfo` from the authenticated `Bundle`; open the bolt connection only when
  `Outcome == Authenticated`. On Rejected/Unreachable/NotAttempted, emit a not-connected
  `ConnectionInfo` carrying the reason (no live bolt attempt) and the existing Error/Warning
  runtime messages (SC-5). `Project` output = `Bundle.Project` (falls back to
  `default-project`).
- `Message`: `Connected · Auth OK` on success; `Auth failed` / `Auth unreachable` /
  `Awaiting token` otherwise.

## Task 3 — Non-persistent token param + secrecy (SC-6)

- New `NonPersistentStringParam : Param_String` (DG.Grasshopper) overriding `Write` to
  `PersistentData.Clear()` before `base.Write(writer)` — value usable live, never saved.
- Drop the every-solve `ScrubTokenPersistentData()`; keep the request-key token hashing
  (`HashToken`) so the raw token never enters the dedup key.

## Task 4 — Tests (DG.Tests) + build

- Extend `ConnectorHeartbeatClientTests`: 200 with a full `neo4j` bundle → `Bundle`
  populated with host URI/user/password/database + project; 200 status-only (no `neo4j`) →
  `Bundle == null`, `Outcome == Authenticated` (back-compat with the existing test).
- Keep existing 401/500/network/non-dgc/empty tests green.
- Source-assertion coverage (comment markers, mirrors 824 style): two-input registration,
  new GUID literal, non-persistent param on Token, auth-gated connection.
