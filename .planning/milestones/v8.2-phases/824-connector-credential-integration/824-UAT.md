# Phase 824: CONNECTOR Credential Integration — User Acceptance Tests

These three checks require a live **Rhino 8 / Grasshopper** session plus a running **data-service** — they cannot be automated in the headless test project (DG.Grasshopper is excluded from DG.Tests). Everything else (build, 234 unit tests, source assertions) already passes; see `824-VERIFICATION.md`.

**Prerequisites**
- `docker compose up -d` (data-service reachable at `http://localhost:8000`, or set the CONNECTOR `DataServiceUrl` input accordingly).
- A minted platform token: in the V2 UI Connectors screen, create a credential on the **Grasshopper** connector and copy the `dgc_…` token (shown once).
- Rebuild/reload the DG plugin: `dotnet build DG/DG.sln -c Release` produces `DG.gha`; load it in Grasshopper.

---

## Test 1 — Valid token authenticates the heartbeat (Criterion 2)
1. Drop a CONNECTOR component. Wire a **Panel** containing the `dgc_…` token into the **Token** input (leave `DataServiceUrl` at its default or point it at your data-service).
2. Set **Connect = true**.

**Expect:** the component `Message` reads **`Connected · Auth OK`**; no Error/Warning bubble appears. In the V2 UI Connectors screen the Grasshopper connector's *last connection* updates to "just now" (data-service recorded the heartbeat).

---

## Test 2 — Invalid token & unreachable service give clear, non-blocking feedback (Criterion 3)
1. Replace the Panel contents with a bogus token `dgc_bogus` (or revoke the real credential in the UI, then reconnect). Set **Connect = true**.

   **Expect:** a **red Error** bubble `CONNECTOR: platform token rejected — invalid, revoked, or expired. …`; `Message` shows **`Connected · Auth failed`**; the **Database** and **Project** outputs still populate (the bolt connection is not blocked). No crash, no silent pass.
2. Stop data-service (`docker compose stop data-service`) and reconnect with any token.

   **Expect:** an **orange Warning** bubble `CONNECTOR: could not reach data-service at http://localhost:8000 …`; `Message` shows `… · Auth unreachable`; outputs still populate; no hang beyond ~6 s (the client timeout).

---

## Test 3 — Token is never persisted; old canvases still work (Criteria 4 & 1)
1. Type a token **directly into the Token input** (internalise it) rather than wiring a Panel.

   **Expect:** a **Remark** `CONNECTOR: platform token was internalised and has been cleared for security — wire it from a Panel instead.`
2. Save the canvas as a `.gh`, then open the saved file in a text/hex viewer and search for `dgc_`.

   **Expect:** **no `dgc_` substring** anywhere in the file (the internalised value was scrubbed before serialization).
3. Open a **pre-v8.2 CONNECTOR canvas** (6-input, saved before this phase).

   **Expect:** it opens with the original six inputs still wired; the two new inputs (`DataServiceUrl`, `Token`) appear appended and unconnected; no "missing component" placeholder (GUID unchanged).
