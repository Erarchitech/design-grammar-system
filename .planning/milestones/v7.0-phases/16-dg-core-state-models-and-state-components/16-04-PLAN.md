Create the v2 statePayloadJson serializer (DesignStatePayloadV2Serializer) with the versioned envelope {version, stateId, capturedAtUtc, objStates[], paramStates[], propStates[]}. Write round-trip tests and rejection tests for v1 payloads.

  <action>
    Create a new static class `DesignStatePayloadV2Serializer` in `DG.Core.Serialization` namespace.

    JSON serializer options:
    - Use the same camelCase naming policy as v1: `PropertyNamingPolicy = JsonNamingPolicy.CamelCase`
    - `WriteIndented = false`
    - Follow the same v1 pattern of a static class with static Serialize/Deserialize methods

    DTO structure (private sealed classes inside the serializer, mimicking the v1 SnapshotDto/ParameterDto pattern):

    ```csharp
    private sealed class DesignStatePayloadV2Dto
    {
        public string Version { get; init; } = "2";
        public string? StateId { get; init; }
        public string? CapturedAtUtc { get; init; }
        public List<ObjStateDto>? ObjStates { get; init; }
        public List<ParamStateDto>? ParamStates { get; init; }
        public List<PropStateDto>? PropStates { get; init; }
    }
    ```

    - ObjStateDto: StateId (string), ObjectRef (string), Label (string?), CapturedAtUtc (string). NO Geometry field.
    - ParamStateDto: StateId (string), CapturedAtUtc (string), Parameters (List&lt;ParameterDto&gt;) — reuse the same ParameterDto pattern as v1
    - PropStateDto: StateId (string), RuleIri (string), DataPropertyIri (string), PropValue (ParameterDto, nullable)
    - ParameterDto: same as v1: ParameterId (string?), DisplayName (string?), Type (string?), Value (object?)

    Serialize(DesignState snapshot):
    - Null check: `ArgumentNullException.ThrowIfNull(snapshot);`
    - Validate: CapturedAtUtc != default, each state in each list has non-empty StateId (defensive check)
    - Sort sub-arrays by StateId string for deterministic output (consistent with D-04 sorted-member hash philosophy)
    - Convert CapturedAtUtc to ISO 8601 round-trip string using the same pattern as v1: `snapshot.CapturedAtUtc.UtcDateTime.ToString("O", CultureInfo.InvariantCulture)`
    - Serialize with JsonSerializer.Serialize

    Deserialize(string json):
    - Null/empty check: throw InvalidOperationException("Design state v2 payload must not be empty.")
    - Parse JSON, wrap JsonException in InvalidOperationException
    - Version check: if dto.Version != "2", throw InvalidOperationException("Unsupported state payload version. Expected '2'.") per D-06
    - Validate required fields: StateId, CapturedAtUtc
    - Parse CapturedAtUtc with DateTimeOffset.TryParse using RoundtripKind
    - Construct DesignState with all three lists, mapping each DTO back to model types
    - For each ObjStateDto: construct ObjState with StateId, ObjectRef, Label, CapturedAtUtc. Geometry = null (not serialized per PROJECT.md).
    - For each ParamStateDto: construct ParamState with StateId, CapturedAtUtc, Parameters from ParameterDto list
    - For each PropStateDto: construct PropState with StateId, RuleIri, DataPropertyIri, PropValue (from ParameterDto via the same FromDto pattern as v1)
    - Call ValidateDeserialized (private method checking non-empty StateIds on all entities, valid timestamps)

    ParameterDto ↔ DesignStateParameter conversion:
    - Reuse the same Type token pattern from v1: "number" → DesignStateParameterType.Number, "integer" → Integer, "boolean" → Boolean
    - ToDto and FromDto private methods following v1's pattern exactly (v1's value parsing: ParseNumber, ParseInteger, ParseBoolean through JsonElement)
    - The v1 ParameterDto class is identical in shape; consider making it shared if there's a clean way, but since the classes are private-sealed inside each serializer, duplication is acceptable per D-07 (new class, clean break)

    Key invariants:
    - The serializer is PURE logic — no GH SDK dependency, no file I/O
    - No reference to v1 serializer classes (DesignStateJsonSerializer, SnapshotDto)
    - Version "2" is hardcoded in the Version property default and checked on deserialization — no version branching
