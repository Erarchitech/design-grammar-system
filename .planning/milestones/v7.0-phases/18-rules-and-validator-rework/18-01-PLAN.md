  <action>
    Create DG/src/DG.Core/Services/DesignStateBindingService.cs as a public static class in namespace DG.Core.Services.

    Single method: `public static List<BindingRow> BuildBindings(Rule rule, DesignState designState)`

    The method follows this algorithm:

    1. **Classify each variable in rule.Variables** using VariableTypeInferrer.Infer(rule, variable.Name).
       - VariableKind.Object → add to objectVars list
       - VariableKind.Property → add to propertyVars list
       - null → check if variable is Builtin-only: scan rule.BodyAtoms + rule.HeadAtoms for BuiltinAtom where the variable appears in Args. If found in any BuiltinAtom and NOT found in any ClassAtom/DataPropertyAtom/ObjectPropertyAtom → exclude from both outputs per D-07. Otherwise → throw InvalidOperationException using ErrorMessageTemplates.RuleVariableUnclassified(rule.Id, variable.Name) per D-06.

    2. **Build BindingRows from ObjStates with Class IRI matching per D-05:**
       - For each Object variable, resolve its target Class IRI from the Rule: find a ClassAtom in rule.BodyAtoms where an Arg with Kind==Variable has Value==objVar.Name. Extract the ClassAtom's PredicateIri (the REFERS_TO Class the variable binds to, e.g. "ex:Building"). Use PredicateIri when non-null; fall back to PredicateLabel or Id as with DataProperty IRIs.
       - For each objState in designState.ObjStates:
         - Check if objState.ClassIri matches the resolved Class IRI for any Object variable. Match via StringComparer.Ordinal.Equals(objState.ClassIri, objectVarClassIri).
         - If objState.ClassIri matches an Object variable's Class IRI: create a BindingRow. For each matching objVar, set ValuesByVar[objVar.Name] = objState.ObjectRef; set ElementRefsByVar[objVar.Name] = new ElementRef { DgEntityId = objState.ObjectRef, DisplayName = objState.Label ?? objState.ObjectRef }
         - If objState.ClassIri does NOT match any Object variable's Class IRI: skip this ObjState (no BindingRow produced). The VALIDATOR will assign ValidStatus=false for this ObjState but exclude it from the overall pass AND per D-02.
         - If objState.ClassIri is null (Class IRI not set on the ObjState): skip (no binding). This handles backward compatibility with existing ObjStates that predate the ClassIri field.

    3. **Apply Property values** from PropStates onto existing rows:
       - For each propState in designState.PropStates:
         - Match by: propState.RuleIri == rule.Id (or rule.Iri — compare with rule.Id using StringComparer.Ordinal)
         - For each propVar in propertyVars:
           - Resolve the DataProperty IRI from the Rule: find a DataPropertyAtom in rule.BodyAtoms where an Arg with Kind==Variable has Value==propVar.Name, and extract the atom's iri/predicateIri
           - If propState.DataPropertyIri matches that IRI → set ValuesByVar[propVar.Name] on ALL existing BindingRows to propState.PropValue?.NumberValue ?? propState.PropValue?.IntegerValue ?? propState.PropValue?.BooleanValue ?? (object?)null

    4. **Handle no-ObjState case**: If objectVars.Count > 0 && designState.ObjStates.Count == 0, throw InvalidOperationException using ErrorMessageTemplates.BindingServiceNoObjectBindings(rule.Id). This prevents silent zero-row evaluation when Object variables have no instances.

    5. **Handle Property-only rule with no ObjStates**: If objectVars.Count == 0 && propertyVars.Count > 0, create a single BindingRow with resolved property values from matching PropStates.

    The DesignState.ObjStates list ordering must be preserved — ValidStatus[i] in the VALIDATOR corresponds to ObjStates[i]. Do NOT sort or reorder ObjStates during iteration.

    Use StringComparer.Ordinal for all dictionary key comparisons (matching existing codebase convention in BindingRow.cs).

    Do NOT add using DG.Core.Classification — that namespace is being deleted (D-04). The old VariableBinder, ClassificationResult, and BindingContext are not referenced.

    Helper method for DataProperty IRI resolution: `private static string? GetDataPropertyIri(Rule rule, string variableName)` — searches rule.BodyAtoms for DataPropertyAtom (Type == "DataPropertyAtom") where an Arg with Kind==Variable has Value==variableName. Returns Atom.PredicateIri if found. The DataProperty IRI is on the atom.predicateIri field (not the iri field per se — check Atom model: PredicateIri vs PredicateLabel vs Id). Use PredicateIri when non-null; fall back to PredicateLabel or Id if PredicateIri is null.
  </action>
