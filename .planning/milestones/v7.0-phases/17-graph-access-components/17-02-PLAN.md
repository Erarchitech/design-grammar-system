    **Neo4jRuleRepository.cs — implement GetObjectsAsync:**
    
    Add a new private const string for the Cypher query (follow the existing `RulesQuery`/`AtomsQuery` naming convention):
    ```csharp
    private const string ObjectsQuery = """
        MATCH (r:Rule {graph:'Metagraph', project:$project})-[:HAS_BODY|HAS_HEAD]->(a:Atom)-[:REFERS_TO]->(c:Class)
        RETURN DISTINCT c.iri AS iri, c.label AS label
        ORDER BY c.label
        """;
    ```

    The query follows D-02: direct REFERS_TO->Class traversal from rules through atoms. DISTINCT eliminates duplicate Classes referenced by multiple rules or multiple atoms within the same rule. ORDER BY c.label gives deterministic ordering for index-matching.

    Implementation of GetObjectsAsync:
    ```csharp
    public async Task<IReadOnlyList<OntologyClass>> GetObjectsAsync(
        ConnectionInfo connection, CancellationToken cancellationToken = default)
    {
        await using var driver = GraphDatabase.Driver(
            connection.Uri, AuthTokens.Basic(connection.User, connection.Password));
        await using var session = driver.AsyncSession(
            options => options.WithDatabase(connection.Database));

        var cursor = await session
            .RunAsync(ObjectsQuery, new { project = connection.Project })
            .WaitAsync(QueryTimeout, cancellationToken);

        var objects = new List<OntologyClass>();
        await cursor
            .ForEachAsync(record =>
            {
                objects.Add(new OntologyClass
                {
                    Iri = record["iri"].As<string>(),
                    Label = record["label"].As<string>(),
                });
            })
            .WaitAsync(QueryTimeout, cancellationToken);

        return objects;
    }
    ```

    Follow the exact driver/session creation pattern from `GetRulesAsync`:
    - `await using var driver = GraphDatabase.Driver(...)`
    - `await using var session = driver.AsyncSession(...)`
    - `session.RunAsync(query, params).WaitAsync(QueryTimeout, cancellationToken)`
    - `cursor.ForEachAsync(...).WaitAsync(QueryTimeout, cancellationToken)`

    Add `using DG.Core.Models;` if needed (likely already present for Rule/OntologyClass).

    The `QueryTimeout` constant (`TimeSpan.FromSeconds(20)`) is already private static in Neo4jRuleRepository — no change needed.
