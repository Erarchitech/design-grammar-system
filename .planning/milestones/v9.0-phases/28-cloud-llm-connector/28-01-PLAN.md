<objective>
Build the core LLM gateway: a provider-agnostic `POST /llm/generate` endpoint in data-service with adapters for Anthropic Claude, OpenAI-compatible APIs, and local Ollama, plus encrypted key storage (`/llm/settings` CRUD), model discovery (`/llm/models`), and connection testing (`/llm/settings/test`).

Purpose: Replace 5 direct Ollama HTTP calls from n8n workflows with one gateway contract. The user controls provider, model, and API key from the UI; the gateway resolves the active provider from settings on every call. No key = Ollama fallback.

Output:
- `data-service/llm_gateway.py` -- Provider adapters (Strategy pattern), Fernet encryption utilities, settings persistence (JSON file in DATA_DIR), Pydantic request/response models, seed model lists, active provider resolution
- `data-service/app.py` -- 6 new FastAPI endpoints for gateway, settings CRUD, connection test, model discovery
- `docker-compose.yml` -- LLM_MASTER_SECRET env var for data-service
- `data-service/requirements.txt` -- cryptography, httpx added (verified installed but missing from requirements)
- `data-service/tests/test_llm_gateway.py` -- pytest suite covering adapter routing, fallback, key masking, encryption round-trip, and error mapping
</objective>

    **LLMAdapter Abstract Base Class (Strategy Pattern):**

    - `LLMAdapter(ABC)`: abstract `generate(self, req: GenerateRequest, api_key: str | None) -> GenerateResponse` and `list_models(self, api_key: str | None) -> list[str]`

    **AnthropicAdapter(LLMAdapter):**

    - Constructor: `__init__(self, base_url: str = "https://api.anthropic.com/v1")` (per D-03, no custom URL from user -- base_url is hardcoded to Anthropic's API)
    - `generate()`: Builds Anthropic Messages API request:
      - Header: `x-api-key`, `anthropic-version: "2023-06-01"`, `content-type: application/json`
      - Body: `{"model": req.model, "max_tokens": 4096, "messages": [{"role": "user", "content": req.prompt}]}`
      - If `req.system` is set, add top-level `"system": req.system` (not inside messages)
      - POST to `{base_url}/messages`
      - Extract text from first `content` block where `type == "text"`, field `text`
      - Extract usage: `input_tokens` -> `prompt_tokens`, `output_tokens` -> `completion_tokens`, sum -> `total_tokens`
    - `list_models()`: GET `{base_url}/models`, parse response list of `{id, display_name}`, return `id` strings

    **OpenAIAdapter(LLMAdapter):**

    - Constructor: `__init__(self, base_url: str = "https://api.openai.com/v1")` -- base_url is configurable from settings for OpenAI-compatible APIs
    - `generate()`: Builds Chat Completions API request:
      - Header: `Authorization: Bearer {api_key}`, `content-type: application/json`
      - Messages array: `system` role first (if req.system set), then `user` role with req.prompt
      - Body: `{"model": req.model, "messages": [...], "max_tokens": 4096}`
      - POST to `{base_url}/chat/completions`
      - Extract text from `choices[0].message.content`
      - Usage maps directly: `prompt_tokens`, `completion_tokens`, `total_tokens`
    - `list_models()`: GET `{base_url}/models`, return list of `id` strings

    **OllamaAdapter(LLMAdapter):**

    - Constructor: `__init__(self, base_url: str = "http://ollama:11434")` -- points to the Docker service
    - `generate()`: Builds Ollama Generate API request:
      - No auth header
      - Body: `{"model": req.model, "prompt": req.prompt, "stream": false, "options": {"temperature": 0.1, "num_predict": 4096}}`
      - If `req.system` is set, add top-level `"system": req.system`
      - POST to `{base_url}/api/generate`
      - 300s timeout (Ollama can be slow on first load)
      - Extract text from `response` field
      - Usage: `prompt_eval_count` -> `prompt_tokens`, `eval_count` -> `completion_tokens`, sum -> `total_tokens`
    - `list_models()`: GET `{base_url}/api/tags`, parse `models` array, return `name` strings

    **Adapter Factory:**

    - `get_adapter(provider: str) -> LLMAdapter`: returns instance of correct adapter. Raises `ValueError` for unknown provider. Case-sensitive per D-02.

    **Encryption Utilities (Fernet, per security contribution and RESEARCH Pattern 2):**

    - `_derive_key(master_secret: str) -> bytes`: SHA-256 hash of master_secret, then base64.urlsafe_b64encode of the digest. Deterministic -- same secret always produces same key.
    - `encrypt_value(plaintext: str, master_secret: str) -> str`: create Fernet with derived key, call `encrypt(plaintext.encode()).decode()`
    - `decrypt_value(ciphertext: str, master_secret: str) -> str`: create Fernet with derived key, call `decrypt(ciphertext.encode()).decode()`. Raises `cryptography.fernet.InvalidToken` if key is wrong or data corrupted.

    **Key Masking (D-03 security, per existing mask_token in app.py line 305):**

    - `mask_key(key: str) -> str`: if key length > 12, return `key[:6] + "..." + key[-6:]`. If length between 5-12, return `key[:2] + "..." + key[-2:]`. If length <= 4, return `"****"`. If empty, return `""`.

    **Settings Persistence (following Speckle pattern in app.py lines 279-322, per D-12 no caching):**

    - `LLM_SETTINGS_FILE` constant: set via `DATA_DIR / "llm-settings.json"` where DATA_DIR is imported from app.py or re-declared from env var `DG_DATA_DIR` defaulting to `/app/data`
    - `load_persisted_llm_settings() -> dict[str, str]`: read JSON file, return dict. Handle missing file and JSON decode errors by returning empty dict.
    - `save_persisted_llm_settings(settings: dict[str, str]) -> None`: create DATA_DIR if not exists, write JSON with indent=2. Strip empty values before writing (matching Speckle pattern).
    - `get_llm_settings_response(settings: dict) -> LLMSettingsResponse`: build response object. Decrypt and mask the apiKey. Only return `apiKeyConfigured: bool` and `apiKeyPreview: str`, never raw key per D-03/LLMC-03.

    **Active Provider Resolution (D-07):**

    - `resolve_active_provider(settings: dict) -> tuple[str, str, str | None]`: returns `(provider, model, api_key)`. If settings has a provider and model and a decrypted apiKey, return them. If any is missing/empty, return `("ollama", None, None)` -- gateway falls back to Ollama with the adapter's default model.

    **Model Discovery (D-13, D-14, D-16):**

    - `_ollama_models_cache: list[str] | None` module-level var for auto-discovered Ollama model list
    - `init_ollama_models()`: called at startup (module import time or init function). GET `http://ollama:11434/api/tags`, extract model `name` values, store in `_ollama_models_cache`. On failure (Ollama not running), set to empty list silently.
    - `list_models_for_provider(provider: str, api_key: str | None, settings_provider: str | None = None) -> list[str]`: If Ollama, return cached models (wrapped in try/except with empty fallback). If Anthropic/OpenAI with valid key, call adapter's `list_models()`. If no key, return `SEED_MODELS[provider]` per D-14.
    - `should_refresh_on_test(provider: str) -> bool`: returns True for anthropic/openai (seed models replaced after successful test key validation), False for ollama (already auto-discovered).

    **Error Mapping (LLMC-06, D-08):**

    - `map_provider_error(exc: Exception) -> tuple[str, str, str]`: returns `(error_message, hint, error_code)` triple for What+Where+How-to-fix pattern. Map httpx.HTTPStatusError (check 401/403/429/5xx), httpx.TimeoutException, httpx.RequestError. Never include raw API key or provider endpoint details in the error message. Error codes: `PROVIDER_AUTH_FAILED`, `PROVIDER_QUOTA_EXCEEDED`, `PROVIDER_TIMEOUT`, `PROVIDER_UNAVAILABLE`, `PROVIDER_UNKNOWN_ERROR`.

    **Exception handling rules:**
    - All adapter HTTP calls wrapped in try/except
    - Network errors and timeouts: map to structured error, never re-raise raw
    - Fernet InvalidToken on decrypt: return empty settings gracefully (Ollama fallback)
    - Ollama connection failure at startup: silently cache empty list

|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Information Disclosure | llm_gateway.py encrypt/decrypt | HIGH | mitigate | Never log raw API key in encrypt_value/decrypt_value or any adapter method. Log only `mask_key(key)` output. Gate: grep for `api_key` in log statements must be preceded by `mask_key(`. |
| T-01-02 | Information Disclosure | docker-compose.yml | HIGH | mitigate | Master secret uses `${LLM_MASTER_SECRET:-default}` pattern; document that .env file must be in .gitignore. Default value is a placeholder string, not a real secret. |
| T-01-03 | Information Disclosure | app.py POST /llm/generate error handler | HIGH | mitigate | Error messages from `map_provider_error()` must not contain raw key text, provider endpoint URLs, or response body snippets. Verified by test_error_mapping assertion. |
| T-01-04 | Tampering | fernet.py decrypt | MEDIUM | accept | Fernet AES-GCM provides authenticated encryption; tampered ciphertext raises InvalidToken on decrypt, which returns empty settings gracefully (Ollama fallback). |
| T-01-05 | Information Disclosure | data-service/data/llm-settings.json | MEDIUM | accept | File lives inside Docker volume /app/data, accessible only within the data-service container. Key is Fernet-encrypted at rest. |
