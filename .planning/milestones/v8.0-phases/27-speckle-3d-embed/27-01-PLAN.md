    Create ui-v2/src/components/speckle/SpeckleViewport.jsx with this React component:

    Props interface:
    - readToken: string — Speckle read token from the view payload (D-03; NOT stored in UI env vars)
    - resourceUrls: string[] — Speckle resource URLs (typically [view.baseResourceUrl, view.validationResourceUrl] from the view payload, filtered(Boolean))
    - project: string — DG project name for world tree entity mapping
    - runId: string — Validation run ID for world tree entity mapping
    - onEntityClick: (dgEntityId: string) => void — Called when a Speckle object is clicked; routes to ModelScreen's pick(id)
    - onReady: () => void — Called after viewer finishes loading all resources
    - onError: (message: string) => void — Called on initialization failure; ModelScreen uses this to trigger fallback to SVG map (D-06)
    - style?: React.CSSProperties — Optional container styling; defaults to `{ position: "absolute", inset: 0 }`

    Refs (useRef):
    - hostRef: HTMLDivElement — the container div that the Viewer mounts into
    - viewerRef: Viewer | null — the Speckle Viewer instance
    - entityMapRef: Map<string, string[]> — Speckle objectId[] per dgEntityId (mirrors legacy App.jsx entityMapRef pattern)
    - disposedRef: boolean — guard flag to prevent state updates after unmount

    Lifecycle (useEffect keyed on [resourceUrls, readToken, project, runId]):
    1. Guard: if resourceUrls is empty or readToken is missing, call onError("No Speckle resource URLs or token available") and return
    2. Dispose: if viewerRef.current exists, call viewerRef.current.dispose() and set to null; clear hostRef's child nodes
    3. Initialize: create new Viewer(hostRef.current, { ...DefaultViewerParams, verbose: false, showStats: false }); await viewer.init()
    4. Extensions: viewer.createExtension(CameraController); viewer.createExtension(SelectionExtension)
    5. Load resources: for each resourceUrl in resourceUrls, call UrlHelper.getResourceUrls(resourceUrl) to obtain concrete object URLs, then for each URL create a SpeckleLoader(viewer.getWorldTree(), url, readToken) and await viewer.loadObject(loader, true)
    6. Entity mapping: walk the world tree using the same collectValidationObjects algorithm from legacy App.jsx (lines 70-141) — iterate all nodes, find those whose raw metadata contains both validationRunId === runId and dgEntityId, collect descendant geometry IDs per entity, store in entityMapRef
    7. Click handler: viewer.on("object-clicked", handler) where handler extracts the hit's raw.dgEntityId from the first hit's node.model.raw, calls onEntityClick(dgEntityId)
    8. Ready: call onReady()
    9. Error: catch any error during initialization, call onError(error.message)

    Cleanup (useEffect return):
    - Set disposedRef.current = true
    - Dispose viewer: viewerRef.current?.dispose()
    - Clear entityMapRef
    - Remove object-clicked listener (viewer.off if available)

    Resize handling (separate useEffect):
    - Create ResizeObserver on hostRef.current
    - On resize: call viewerRef.current?.resize() and viewerRef.current?.requestRender(UpdateFlags.RENDER | UpdateFlags.SHADOWS)
    - Must guard against missing ResizeObserver (SSR safety)

    Render:
    - A containing div with the passed style (or default absolute-fill)
    - Inside it, a div ref={hostRef} with style `{ width: "100%", height: "100%", position: "absolute", inset: 0 }` — this is the Speckle Viewer mount target
    - No loading overlay in the component itself; ModelScreen handles loading/empty states via onReady/onError

    No FilteringExtension usage per D-04: colors are server-side applied during Speckle publish (speckle_validation.py build_validation_entity sets vertex colors based on pass/fail). The viewer renders the geometry as-is. No client-side recoloring.

    No CameraController.fit() call per D-02: camera resets naturally because a new Viewer instance is created on each run change.

| Boundary | Description |
|----------|-------------|
| Browser -> data-service | Untrusted network; view payload with Speckle read token crosses HTTP |
| @speckle/viewer -> Speckle server | Viewer fetches geometry from Speckle using read token in URL params |
| npm @speckle/viewer install | Third-party package supply chain; runs in Vite build |
| Speckle geometry rendering | Untrusted BIM geometry parsed and rendered in browser WebGL context |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

Same pattern as legacy model-viewer (App.jsx line 714: `m.readToken` passed to SpeckleLoader). Token is scoped to Speckle server, read-only, and scoped to the project's models. No UI-visible env vars per D-03. |
| T-27-02 | Tampering | npm @speckle/viewer dependency | high | mitigate | @speckle/viewer ^2.28.0 is already installed in the legacy model-viewer (graph-viewer/model-viewer/package.json) — same package, same version range. No new supply chain exposure. The legacy viewer has been running in production. Standard npm integrity checks apply. |
| T-27-03 | Tampering | Speckle geometry rendering | low | accept | Speckle Viewer sandboxes geometry rendering within WebGL context.
