# Plan 24-01: Model Viewer — Validation Runs UI

> Documented post facto per milestone directive.

## Objective

Validation results browsable in the V2 skin: run list from data-service (`/validation/runs/{project}`, `/validation/view/{project}/{runId}`), design-state grouping (by `statePayloadJson` summary stateId, per the legacy `useValidationRunsGrouping` convention), failing/passing collapsible groups with instance inspection, stylised isometric map canvas with validation colouring + pan/zoom + minimap, SWRL rule panel resolved from the metagraph, and an orbit-glyph status legend.

## Approach

- **`modelApi.js`** — data-service validation endpoints + targeted Cypher for `Rule.SWRL/RuleName/RuleDescription` and per-entity rule statuses (`ValidationEntity {runId, dgEntityId}`).
- **`ModelScreen.jsx`** — three-column layout per the mockup: left = design-state group tiles + orbit-glyph legend; centre = toolbar (Failing/Passing/Base toggles, Fail/Pass α sliders, Select-by-Id, Isolate/Hide/Show all), SVG iso map with wheel zoom about cursor + drag pan + minimap with viewport rect + click-to-centre, run tile strip; right = Properties sidebar with run mode (KV rows, StatBlocks, collapsibles, rule panel with SWRL CodeBlock) and instance mode (per-rule pass/fail breakdown).
- **Geometry**: entity boxes are deterministic synthetic isometric massing derived from entity-id hashes — the V2 spec deliberately models the viewport as a stylised map; the true Speckle 3D embed is deferred (REQUIREMENTS Future).
- **Fixture**: `test/seed_validation_run.cypher` (project `v8-ui-smoke`, schema v4) because real runs publish only from Grasshopper VALIDATOR and the pre-v4 live runs are pending the graph-tag migration.

## Verification

Live: run `v8smoke-run-001` listed with state group `DS_V8SMOKE01`; failing/passing groups (3/5) with selection; instance mode shows per-rule Fail row via live Cypher; map renders 8 boxes (3 signal-stroked), wheel zoom changes viewBox, minimap present; rule panel shows the seeded SWRL verbatim in mono.
