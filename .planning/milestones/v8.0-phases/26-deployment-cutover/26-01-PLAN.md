# Plan 26-01: Deployment Cutover and E2E Parity

> Documented post facto per milestone directive.

## Objective

V2 is the product UI: the `design-grammars` container serves the `ui-v2` build at :8080 through the existing nginx reverse-proxy routes; the legacy dark SPA is archived with release notes; the E2E parity checklist passes against live services.

## Approach

- `ui-v2/Dockerfile`: node:20-alpine build stage → nginx:alpine; `nginx.conf` keeps the exact legacy proxy routes (`/neo4j`, `/n8n`, `/data-service`, `/llm`); `entrypoint.sh` regenerates `/config.js` (`window.GRAPH_CONFIG`) from environment; `public/config.js` empty default for dev; `index.html` loads `/config.js` before the bundle.
- `docker-compose.yml` `design-grammars.build` → `./ui-v2` (env vars unchanged).
- Legacy `graph-viewer/` archived **in place** (retirement documented in `docs/RELEASE-NOTES-v8.0.md` and CLAUDE.md) — history preserved, `.pen` interface sources remain referenced by the design spec.
- `docker compose build --no-cache design-grammars && up -d`.
