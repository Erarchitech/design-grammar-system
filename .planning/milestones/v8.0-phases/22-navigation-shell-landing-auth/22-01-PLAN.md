# Plan 22-01: Navigation Shell, Landing and Auth

> Documented post facto per milestone directive.

## Objective

The app opens on the V2 landing experience: particle-ring hero with the materialising "DESIGN GRAMMARS." title, region-callout navigation between the four screen layers (520ms scale+opacity fly transitions), and inline login/register on the landing via the rising frost card, with persistent session state.

## Context inputs

- `design/v2/Design Grammars V2.dc.html` lines 31-86 (landing markup), 498-520 (state), 826-1132 (lifecycle/navigation/particles/hero), 1889-2005 (auth handlers) — the mockup's `data-dc-script` logic, ported nearly verbatim
- `graph-viewer/index.html` RegisterPage (~1164-1201) — legacy auth store contract

## Tasks

1. **Landing engine + layer** — `landingEngine.js` (canvas particle ring with gaussian anchor peaks, leader-line callouts, hero materialise/dissolve phases, region-label layout); `LandingLayer.jsx` (hero DOM, guest CTA / member state, auth frost card); `auth.js`
2. **Shell** — `App.jsx` region state + `Layer` wrapper (active scale 1, landing recedes at 1.6 toward the clicked callout via transform-origin, others 0.965); `ScreenHeader.jsx` global chrome; placeholder Graph/Model/Projects screens; Escape returns to landing

## Key design points

- Engine-animated DOM (hero title, beneath, region labels) styled ONLY by the engine; React renders classes, so re-renders can't clobber per-frame values (`landing.css`)
- Auth is byte-compatible with the legacy store: `dg_users` (salted SHA-256 `dg_salt_` + password, hex) + `dg_current_user` — existing accounts survive the Phase 26 cutover
- Register takes a single "Full name" field (per spec) split into legacy name/surname
- Selected project persisted at `dgv2_project` (consumed in Phase 25)

## Verification

Browser (Vite dev, live checks): hero forms from particles after fonts load; region labels navigate with landing transform-origin at the ring point; chrome title + mono subtitle on non-landing screens; Escape and ← Back return to landing with hero re-forming; register→member state→sign out round-trip with `dg_users`/`dg_current_user` correctly written; auth errors inline in Signal Red.
