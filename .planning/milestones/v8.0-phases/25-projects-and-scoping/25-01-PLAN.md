# Plan 25-01: Projects and Scoping

> Documented post facto per milestone directive.

## Objective

Projects are first-class: the Projects screen lists real projects from Neo4j (`DISTINCT n.project` + node counts) as V2 tiles; opening one persists it (`dgv2_project`) and scopes every Graph Viewer and Model Viewer query; New Project sets a fresh scope that materialises in the DB on first ingest.

## Approach

Scoping was architected in from Phase 22: `project` state lives in `App.jsx`, persists to localStorage, and flows into every screen; Graph/Model data loaders re-fetch on change. This phase adds the live tile grid, the New Project inline flow, and (mockup behaviour) returning to the landing after opening a project — where the member state shows the active project next to the red status dot.

## Verification

Live: 8 real projects rendered as tiles with node counts; opening `URBAN_BLOCK_V7` re-scoped the datascape to exactly its data (OntoGraph:6 / Metagraph:22 / SpecGraph:2, matching direct DB counts) and the chrome subtitle; switching to `v8-ui-smoke` re-scoped the Model Viewer to its run.
