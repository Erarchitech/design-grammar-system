Create `n8n/workflows/knowledge-query.json` as a valid n8n workflow JSON following the structure of `graph-query-mcp.json`.

**Workflow metadata:**
- `"name": "Knowledge Query"`
- `"active": true`

**Webhook node:**
- `"path": "dg/knowledge-query"` (per D-09)
- `"webhookId": "dg-knowledge-query"` (unique, per Pitfall 4)
- `"httpMethod": "POST"`
- `"responseMode": "responseNode"` (enables ack path)

**Node sequence:**

1. **Webhook** — POST `/dg/knowledge-query`, expects `{"prompt_text": "...", "project_name": "..."}`

2. **Set Input Defaults** (Set node) — Set `prompt_text` = `{{$json.body.prompt_text}}`, `project_name` = `{{$json.body.project_name}}`

3. **Format Ack** (Set node) — Set response body: `{"status": "accepted", "executionId": "{{$execution.id}}"}`

4. **Respond Ack** (Respond to Webhook node) — Returns ack JSON immediately

5. **Build Full-Text Cypher** (Function node) — JavaScript code that:
   - Reads `$json.prompt_text` and `$json.project_name`
   - Sanitizes the prompt: strips Lucene special chars using `/[+\-!(){}\[\]^"~*?:\\/]/g` replacement with space (per Pitfall 2)
   - If cleaned query is empty after sanitization, falls back to `"*"`
   - Builds the deterministic full-text search Cypher (NOT LLM-generated — per D-06):
   ```
   CALL db.index.fulltext.queryNodes('knowledge_note_search', $query) YIELD node, score WHERE node.project = $project AND node.graph = 'KnowledgeGraph' RETURN node.noteId AS noteId, node.title AS title, node.content AS content, score ORDER BY score DESC LIMIT 5
   ```
   - Saves the Cypher string in `$json.cypherUsed` for QRYK-02
   - Saves the cleaned query in `$json.cleanedQuery`
   - Returns `{ json: { ...input, cypherUsed, cleanedQuery, neo4jBody: { statements: [{ statement: cypher, parameters: { query: cleanedQuery, project: project_name } }] } } }`

6. **Execute Full-Text Search** (HTTP Request node) — POST to `http://neo4j:7474/db/neo4j/tx/commit` with basicAuth `neo4j:12345678`. Body: `={{JSON.stringify($json.neo4jBody)}}`. Content-Type: `application/json`.

7. **Build Answer Prompt** (Function node) — JavaScript code that:
   - Reads Neo4j response `$json.results[0].data` array
   - Extracts top 5 results (per D-05): title and content from each row
   - Counts matches: `matchCount = data.length`
   - If no results, builds prompt: `"No matching knowledge notes found for project '${project}'. Reply: 'No matching knowledge notes found for this project.'"`
   - If results exist, builds prompt:
   ```
   You are an architectural knowledge assistant. Answer the question using ONLY the provided notes.
   If the notes don't contain enough information, say so.

   Question: {prompt_text}

   Notes:
   1. [{title}]: {content}
   2. [{title}]: {content}
   ...

   Answer concisely in 1-3 sentences.
   ```
   - Returns `{ json: { ...input, matchCount, answerPrompt } }`

8. **Ollama Generate Answer** (HTTP Request node) — POST to `http://ollama:11434/api/generate` with body:
   ```json
   {"model": "llama3.1:latest", "prompt": "={{$json.answerPrompt}}", "stream": false, "options": {"temperature": 0.3, "num_predict": 512}}
   ```
   NOTE: No `"format": "json"` here — answer is free-text NL, not JSON. Temperature 0.3 (slightly higher than insert for natural language). Timeout: 900000ms.

9. **Parse Answer** (Function node) — JavaScript code that:
   - Reads `$json.response` from Ollama
   - Trims whitespace
   - If empty, sets answer to `"No answer could be generated."`
   - Returns `{ json: { ...input, answer: trimmedResponse } }`

10. **Write KnowledgeSession** (HTTP Request node) — POST to `http://neo4j:7474/db/neo4j/tx/commit` with basicAuth. Body is a statements array with one MERGE statement for KnowledgeSession:
    - `sessionId`: compute as `'ks-' + Date.now().toString(36)` in a preceding Code/Set node or inline expression
    - `mode`: `'query'`
    - `prompt`: `$json.prompt_text` (parameterized, not interpolated — per T-3-02)
    - `result`: `JSON.stringify({answer: $json.answer, cypherUsed: $json.cypherUsed, matchCount: $json.matchCount})` truncated to 2000 chars
    - `project`: `$json.project_name`
    - `graph`: `'KnowledgeGraph'` (hardcoded — per T-3-01)

11. **Post Execution Result** (HTTP Request node) — POST to `http://data-service:8000/execution-result` with body:
    ```json
    {
      "executionId": "={{$execution.id}}",
      "status": "completed",
      "workflow": "knowledge-query",
      "payload": {"answer": "={{$json.answer}}", "cypher": "={{$json.cypherUsed}}", "matchCount": "={{$json.matchCount}}"}
    }
    ```
    NOTE: payload key is `"cypher"` (not `"cypherUsed"`) to match existing UI polling pattern in `graph-viewer/index.html` line 1631 which reads `payload.cypher`. Workflow key MUST be `"knowledge-query"` (hyphen — per D-08, Pitfall 3).

**Connection wiring:**
- Webhook -> Set Input Defaults -> Format Ack -> Respond Ack (ack path ends here)
- Set Input Defaults -> Build Full-Text Cypher -> Execute Full-Text Search -> Build Answer Prompt -> Ollama Generate Answer -> Parse Answer -> Write KnowledgeSession -> Post Execution Result (work path)

**Critical details:**
- The full-text search Cypher is deterministic — built in a Function node, NOT by LLM (per D-06)
- ALL Cypher uses parameterized queries ($query, $project) — prompt_text NEVER interpolated into Cypher (per T-3-02)
- `graph` parameter hardcoded as `'KnowledgeGraph'` (per T-3-01)
- Lucene special characters sanitized from query text before sending to Neo4j (per Pitfall 2)
- Workflow key is `"knowledge-query"` with hyphen (per Pitfall 3)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser -> n8n webhook | Untrusted NL question enters as `prompt_text` |
| n8n -> Neo4j full-text search | Sanitized query text used as Lucene query parameter |
| n8n -> Ollama | Search results + question forwarded for answer synthesis |
| n8n -> data-service | Execution result payload posted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Tampering | n8n Build Full-Text Cypher node | mitigate | Hardcode `graph: 'KnowledgeGraph'` in Cypher parameters — never from user input. Prevents cross-graph reads. |
| T-3-02 | Injection | n8n Neo4j HTTP calls | mitigate | Cypher uses parameterized $query and $project — prompt_text NEVER interpolated into Cypher. Lucene special chars sanitized before use as full-text query parameter. |
| T-3-03 | Information Disclosure | Ollama answer prompt | accept | Search results from same project only (WHERE node.project = $project). Answer based on user's own project data. Local Ollama. |
| T-3-06 | Injection | Lucene query syntax | mitigate | Sanitize Lucene special characters from prompt_text before passing as $query parameter. Regex strips `+`, `-`, `!`, `(`, `)`, `{`, `}`, `[`, `]`, `^`, `"`, `~`, `*`, `?`, `:`, `\`, `/`. |
</threat_model>
