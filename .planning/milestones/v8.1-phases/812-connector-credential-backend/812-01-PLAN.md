# Plan 812-01: Connector Credential Backend

## Objective

data-service can mint, list, and revoke connector credentials and track per-connector connection status via a token-authenticated heartbeat endpoint. This is the platform side of the connector ecosystem; UI comes in Phase 813, docs in 815.

## Context (read before coding)

- `data-service/llm_gateway.py` — the storage + crypto pattern to mirror: Fernet key derived via SHA-256 from a master secret (`_derive_key`/`encrypt_value`), settings persisted as JSON under `data-service/data/` (`llm-settings.json`), loaded/saved via small helpers. Follow this style; do not invent a new persistence layer.
- `data-service/app.py` — FastAPI app; `/llm/settings` endpoints (~line 912) show the endpoint + pydantic payload conventions and structured JSON error responses.
- `data-service/tests/test_llm_gateway.py` — test conventions (pytest, FastAPI TestClient or direct function tests).

## Tasks

### Task 1: `data-service/connectors.py` module

- `CONNECTOR_REGISTRY`: the 13 connectors / 5 categories above (id, name, category).
- Credential model: `{credential_id, connector_id, label, token_hash, created_at, revoked, last_connection}`. Tokens: `dgc_` + `secrets.token_urlsafe(32)`, generated server-side, returned **once** on creation; only SHA-256 `token_hash` is persisted (hashing beats encryption here — we never need the plaintext back).
- Persistence: `data/connector-credentials.json` via load/save helpers mirroring llm_gateway's settings persistence.
- Status derivation per connector: `never_connected` (no successful heartbeat ever), `active` (last connection ≤ 7 days ago), `stale` (older). Expose `last_connection` ISO timestamp.

### Task 2: endpoints in `app.py`

- `GET /connectors` — registry joined with per-connector status + last-connection date + credential summaries (id, label, created_at, revoked; never token or hash) (CONNB-01, CONNB-04)
- `POST /connectors/{connector_id}/credentials` — body `{label?}`; 404 unknown connector; returns `{credential_id, token}` — the only time the token is visible (CONNB-02)
- `DELETE /connectors/{connector_id}/credentials/{credential_id}` — revoke; revoked tokens must stop authenticating (CONNB-01)
- `POST /connectors/heartbeat` — `Authorization: Bearer dgc_...`; matches hash against non-revoked credentials; updates that connector's `last_connection`; returns `{connector_id, status}`; 401 on unknown/revoked token (CONNB-03)

Structured JSON errors matching existing app.py conventions.

### Task 3: `tests/test_connectors.py`

Cover: registry shape (13/5); create returns token once and persists only the hash; heartbeat with valid token flips status to `active` and stamps `last_connection`; heartbeat with bad/revoked token → 401; status derivation for never_connected/active/stale (freeze/patch time for stale); revoke lifecycle.

### Task 4: Run the suite

`python -m pytest data-service/tests/ -q` — all green (pre-existing failures, if any, noted but not in scope).
