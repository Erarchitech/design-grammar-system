# Plan 810-01: Ring Extension & Screen Shell

## Objective

The landing particle ring gains four new region callouts — **AI Engine**, **Connectors**, **Reasoner**, **DG API Docs** — alongside Graph Viewer / Model Viewer / Projects. Each flies into its own screen layer (skeletal for now; filled by phases 811/813/814/815) with the standard 520ms transition and back navigation.

## Context (read before coding)

- `ui-v2/src/landing/landingEngine.js` — `ANCH` array at top (`{key, ang, amp, sig}` per region; angles in radians, screen coords so +y is down: `graph` ≈ π·1.06 (left), `model` ≈ -0.2 (right), `projects` ≈ π·0.62 (lower-left)). `layoutLanding()` has a `cfg` map keyed by the same keys (label element + `out` offset); `pk[key]` becomes the fly-origin. `setNavLabels()` iterates the label elements explicitly.
- `ui-v2/src/landing/LandingLayer.jsx` — one ref + one `.dgl-region-label` div per region; `labels: {graph, model, projects}` passed to the engine; `fly(region)` calls `onFly(region, engine.pk[region])`.
- `ui-v2/src/App.jsx` — `region` state + one `<Layer>` per screen; new screens follow the `ProjectsScreen` prop pattern (`active`, `onBack`, `project`).
- `ui-v2/src/screens/ProjectsScreen.jsx` — copy its chrome/back-navigation pattern for the skeletons.

## Fixed contract (downstream phases depend on these exact names)

- Region keys: `aiengine`, `connectors`, `reasoner`, `apidocs`
- Screen files: `ui-v2/src/screens/AiEngineScreen.jsx`, `ConnectorsScreen.jsx`, `ReasonerScreen.jsx`, `ApiDocsScreen.jsx`
- Callout label text: `AI Engine`, `Connectors`, `Reasoner`, `DG API Docs`

## Tasks

### Task 1: Engine — four new ring anchors

Add 4 `ANCH` entries with angles chosen so all 7 callouts + the hero title + the hint stay clear of each other at common desktop sizes (e.g. distribute in the free arcs: top-right ~-π·0.45, top-left ~-π·0.78, lower-right ~π·0.30, left ~π·0.86 — tune visually). Use amp/sig in the existing range (amp 0.6–0.8, sig 0.2–0.26) so new gaussian peaks read as secondary to the three primary regions. Extend the `cfg` map in `layoutLanding()` (out: 54) and make `setNavLabels()` iterate all label elements generically (`Object.values(this.els.labels)`).

### Task 2: LandingLayer — four new callouts

Add refs + `.dgl-region-label` divs for the four regions (same class, `onClick={() => fly("<key>")}`), and pass them in the `labels` record to the engine.

### Task 3: Skeleton screens + App layers

Create the four screen components: full-viewport container, V2 chrome header with back navigation (mirror ProjectsScreen), `dg-annotation` region eyebrow, screen title, and a `dg-frost` placeholder card stating the region's purpose (one sentence each). Register four `<Layer>` entries in App.jsx with region keys per the contract. Escape-to-landing must work (it's generic on `region !== "landing"` — verify).

### Task 4: Verify

`npm --prefix ui-v2 run build` passes. Start dev server (`npm --prefix ui-v2 run dev`), confirm: 7 callouts visible without overlap, each new callout flies in/out correctly, existing 3 regions + auth card unaffected.
