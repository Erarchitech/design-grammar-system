# Milestone Seed: v10.0 — Script Intelligence

**Seeded:** 2026-07-08 (during the v9.0 restructure)
**Status:** CONSUMED 2026-07-08 — elaborated into [v10.0-REQUIREMENTS.md](v10.0-REQUIREMENTS.md) + [v10.0-ROADMAP.md](v10.0-ROADMAP.md) (isolated future milestone; both candidate capabilities confirmed in scope). Kept for provenance; the requirements/roadmap files are authoritative.
**Builds on:** v9.0 Phases 5–10 (Computgraph serialization pipeline, DG Canvas Bridge, structure validation MVP)

## Core value

The connected LLM becomes an active partner *on the Grasshopper canvas*: generating parts of scripts, editing parts of scripts, and consulting the architect — all grounded in the Computgraph, Design Rules mapped to script structure, and the DG Canvas Annotation Convention established in v9.0.

## Capability seeds

1. **Generate parts of scripts** — the bridge gains write commands per the grasshopper-mcp reference (`add_component`, `connect_components`, `validate_connection`, component knowledge base, pattern creation); generation targets a tagged Procedure/Pattern slot; every generated block arrives as a *preview* (the v9.0 proposal style) and requires on-canvas confirmation, mirroring RCGN-04.
2. **Edit parts of scripts** — old→new diff preview on the canvas (analogous to the v9.0 RING-02 atom-diff for rules): proposed rewires/replacements rendered as preview groups + ghost wires; confirm applies inside an undo record.
3. **Consulting assistant** — full conversational surface over `POST /computgraph/consult` (v9.0 Phase 10 seed): multi-turn, rule-aware ("does this script satisfy R_URB_HEIGHT_MAX_75_V structurally?"), able to reference live canvas state via the bridge and published Computgraph history.
4. **Structure-rule vocabulary growth** — the v9.0 `requiresProcedure/requiresParameter` mapping grows into a proper grammar of structural constraints (cardinality, topology, parameter-domain rules), possibly graph-native (rules stored as Metagraph extensions rather than a JSON file).
5. **Candidate: cluster introspection** — Procedures as GH Clusters (opaque in v9.0) opened up for recognition.
6. **Candidate: granular wire persistence** — persist `CgNode`/`CgWire` substrate per-node in Neo4j (v9.0 stores the full `cgContextJson` blob on `Algorithm`) to enable topology queries and diffing.
